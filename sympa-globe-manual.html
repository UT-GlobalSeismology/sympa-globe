Sympa! Globe ユーザーマニュアル

Sympa! Globe は、ブラウザだけで動く単一 HTML の世界地図・地球儀ツール。プレート運動・海洋底拡大・地震分布・マントルの三次元構造などを、CLI コマンドで自在に切り替えながら見られる。

1. クイックスタート

map proj ortho              # 地球儀表示に切替
map plate-model merdith2021 # 大陸復元モデルを選択
map plate-age 200           # 200 Ma（恐竜時代)の大陸配置に
map tint age                # 海洋底年代を色で表示
map split age relief        # 年代と地形を左右に並べて比較
share qr                    # 今のビューを QR コードで共有（教室提示・再現用）

  • ⚠ share／share qr はコマンド操作のみを再現する。マウスのドラッグ回転・パン・ホイールズームは共有リンクに含まれないので、見せたい向き・倍率は map rotate 等のコマンドで指定してから共有する（§ 4.42）。

  • help でコマンド一覧、map でいつでも地図に戻れる。

  • save fig.svg / save fig.png で現在のビューを論文・スライド用にベクター/ラスター出力できる。

  • 起動するとそのまま地図モードで立ち上がる（共有リンク #s=… で開いた場合はそのビューを再現）。

起動

open sympa-globe.html

キーボード

  • Tab: コマンド・ファイル名補完
  • ↑ / ↓: 履歴
  • Ctrl-L または clear: コンソールクリア
  • CLI 上端をドラッグで高さ調整

コマンドの連結（;・複数行）

複数のコマンドを セミコロン ; で区切って一行に並べると、左から順に実行される:

map proj ortho; map tint age; map plate-model Merdith2021; map plate-age 200

改行で区切った複数行をまとめて貼り付け（ペースト）しても同様に順次実行される（; と改行は混在可）。# や // で始まる行はコメントとして読み飛ばすので、注釈つきの手順ブロックをそのまま貼り付けられる。↑ で履歴を辿ると連結した一行全体が一度に呼び出される。

4. 世界地図モード map

震源・観測点を世界地図上に配置し、震源↔︎観測点の大円弧を描画するモード。recsec 同様プロットエリアを地図に切り替える。

4.1. 入る・出る

map                     # 地図モード ON（波形プロットを地図に切替）
map off                 # 通常プロットへ戻る（traces 数に応じて single / p1）

r でも rh（ヘッダのみ）でも動く。地理 field（EVLA/EVLO/EVDP/STLA/STLO）が入っていれば、データ部は不要。

4.2. 提示モード（文字・線を大きく／太く）map display

教室でスクリーンに投影すると、既定の文字サイズ・線幅では見えづらいことがある（実授業のフィードバック）。提示スケールで地図の 文字・線・マーカーをまとめて拡大 できる（地図そのものの位置・範囲は不変＝同じ地図のまま視認性だけ上げる）。

map display              # 現在の提示スケールを表示
map display l            # 1.5×（s=標準1 / l=1.5 / xl=2 / xxl=3）
map display xl           # 2×
map display 1.8          # 数値で 1〜4 倍を直接指定
map display s            # 1×（標準に戻す）
  • 拡大対象＝地図の ラベル文字・グリッド度数・海岸線/プレート境界などの線・震源★/観測点▽等のマーカー。上部の状態表示（HUD）は重ならないよう拡大しない。
  • 提示スケール（map display）は画面表示のみで、SVG/PNG 書き出し（map save）は標準サイズのまま。論文・印刷用に書き出しだけ拡大したいときは map paper（下記 5.3）を使う＝画面と書き出しでスケールを分離できる。
  • iPad 等でも Web として表示は可能（タッチ操作の最適化は今後の課題）。

4.3. 論文・印刷モード（書き出しだけ拡大）map paper

map display が画面表示を拡大するのに対し、map paper は map save で書き出す図だけ 文字・観測点記号・凡例（カラーバー）・震源記号を一括拡大する。画面表示（map display）は変えない＝画面は普段どおり、論文図だけ印刷サイズで読める大きさに（display と paper の分離）。論文では図を段組幅（例 0.85\linewidth）に縮小配置するため、既定サイズのままだと文字・記号が小さくなりすぎる問題への対処。

map paper              # 現在の書き出し用スケールを表示
map paper 1.8          # map save 書き出し時のみ 1.8×（1〜4倍）
map paper off          # 等倍で書き出し（既定）
  • 拡大対象＝ラベル文字・グリッド度数・観測点(sites)ドットと群凡例・スカラー場カラーバー（箱＋目盛）・震源★/ビーチボール。地図の投影・座標・データ配置は不変（注記だけ拡大）。
  • スカラー場（map scalar）のカラーバーが左上を占める図では、sites 群凡例を自動で左下へ退避＝左上角の観測点（例 FKU）と重ならない。凡例文字は白縁取り（halo）で地図上でも読める。
  • 典型ワークフロー: 画面で構図を決める → map paper 1.8 → map save fig.png 1100 1000（→ 論文へ）。map display（画面拡大）とは独立に効く。

4.4. 小学生モード（ふりがな）map ruby

地図の日本語ラベル（プレート名・地形・用語）に ふりがな を振る。map ruby on で小学生でも読めるように。canvas はルビ非対応なので、漢字の上に小さくふりがなを描く。読みの辞書に載っている語だけ 振る（読みが不確かな語は振らない＝捏造しない）。カタカナ名（ユーラシア等）は既に読めるので対象外。

map ruby on             # ふりがな ON（小学生モード）
map ruby off            # OFF（既定）
  • 対応語＝プレート/大陸名（太平洋→たいへいよう・北米→ほくべい・南極→なんきょく・フィリピン海の「海」→かい 等）と一般用語（地質・海洋・海溝・海嶺・中央・造山・年代・現在・日本 等）。未対応語は振らない（今後辞書を追加可）。
  • map display（提示モード）と併用可＝ふりがなも一緒に拡大。
  • SVG 書き出し（map save）はふりがな無し（英語ラベルで出力）。

4.5. 投影法（拡張版）

map proj plate-carree          # 既定。equirectangular
map proj aeqd                  # 中心点周りの等距離（震源中心）
map proj ortho                 # 球面（半球見える）
map proj mollweide             # 楕円形 pseudocylindrical 等積

map rotate <lon> <lat>         # Orthographic の中心経緯度
map rotate auto                # [0, 0] にリセット
map home                       # 視点リセット（回転→[0,0]＋ロック解除、ズームは維持）
map lock <地塊名>              # 指定地塊を画面中央に固定（プレート復元＋ortho 専用）
map lock off                   # ロック解除
map height <vh|px|auto>        # 地図キャンバスの高さ（例 90=90vh / 700=700px / auto=自動）

地図モードでは上部の「ファイル選択」欄を自動で隠し、キャンバスを CLI の上を埋める高さ （画面の縦幅と CLI の高さに連動）にして 地球儀（ortho）を大きく表示する。もっと大きく/小さく したいときは map height 90（90vh）や map height 700（700px）、auto で自動に戻す。ブラウザを 全画面（F11）にすると更に広がる。map off で波形表示に戻ると欄も戻る。

Orthographic（globe）: - 視点の中心を state.map.rotate = [lon, lat] で指定（default [0, 0]） - 裏側半球は描画されない（project() が null） - 左ドラッグでグローブ回転（Google Earth 風）。右ドラッグはパン - ホイールでズーム（半球を拡大） - 大円弧は disc 境界で自動切断、震源と観測点が裏側なら表示されない - 右上の ⟲ ボタン（ortho 時のみ表示）で視点リセット（map home と同じ＝回転を [0,0] に戻し、ロックも解除）。ズーム/パンは維持される（それらはダブルクリックで auto-fit） - 視点ロック（地塊追従）: プレート復元中（map plate-model）に地球上の地塊をクリックすると、その地塊を画面中央に固定。タイムスライダー（map plate-age）を動かすと中心がその地塊に追従して移動するので、注目地塊（例: 南中国）を見つめたまま時代を遡れる。CLI は map lock 南中国、地塊名はモデルのラベル一覧に準拠。同じ地塊を再クリック／⟲ ボタン／map home で解除。ロック中は右上に 🔒 <地塊名> バッジ

Mollweide: - 世界全体が楕円に収まる、等積投影（プレート相対面積が保たれる） - 経度 ±180° で seam あり（中央子午線に対する相対位置） - map rotate <lon> で中央子午線を移動できる（Pacific 中心 / Africa 中心など）。極軸は固定（縦のまま）。latitude 成分は無視される（極軸を倒すと seam invariant が壊れるため、3 軸回転は Ortho の責務） - 左ドラッグでも経度方向の回転できる（Ortho と統一感、上下方向のドラッグは無視）

プレート運動ベクトルは 全 4 投影で描画される。各点で local east/north Jacobian を計算して投影歪に対応。

4.6. 投影法

map proj plate-carree   # 等経等緯（既定。複数震源 OK）
map proj aeqd           # 方位等距離（中心震源、大円が直線）
map proj                # 現在の投影法を表示

aeqd は単一の中心点を必要とするため、map.events が複数イベントを含むときは自動的に plate-carree で描画される（設定 state.map.proj 自体は維持）。

map center <lon> <lat>  # AEQD 中心を明示指定
map center auto         # 中心指定を解除（先頭イベントに戻る）

4.7. レイヤと精度

map layer coastline on|off    # 海岸線（NE 50m, 既定）の ON/OFF
map coast load <file>         # 自前の細かい海岸線(TopoJSON {transform,arcs})に差替（off で内蔵へ復帰）
map coast land <file>         # 閉じた陸ポリゴンを白で上塗り(海底色が海岸線を越えないように。off で解除)
map layer bathymetry on|off   # 海底地形（青の深度濃淡）の ON/OFF（既定 OFF）
map layer graticule on|off    # 緯度経度グリッド + 枠の度ラベルの ON/OFF（既定 ON）
map layer raypath on|off      # 大円弧の ON/OFF（震源-観測点直結）
map layer plate on|off        # プレート境界（Bird 2003 PB2002）の ON/OFF
map layer quakes on|off       # 世界の震源 overlay（USGS M≥5.5）の ON/OFF（既定 OFF）
map layer volcanoes on|off    # 完新世の火山（GVP、三角）の ON/OFF（既定 OFF）
map style volcano-class <設定>  # 火山をテクトニックセッティングで絞る
map precision coarse|accurate # 大円補間密度（32 / 128 点）
map contour <deg>             # 震央距離コンター（同心円）を <deg>° おきに描画
map contour off               # コンターを消す
map save <path>.svg           # 現在のビューを SVG ベクター画像で保存
save <name>.svg|.png [w h [scale]]  # 現在のビューを保存。.png は実 PNG（ブラウザは SVG をラスタ化）

save <name>.png 1200 1100 のように 拡張子で SVG/PNG を選べる（.png 指定でブラウザは内部で SVG をラスタライズして実 PNG を出力。ヘッドレス render.js は resvg を使用）。.svg または拡張子なしはベクター SVG。PNG は既定で 2×（画面の DPR）で高精細にラスタ化する。もっと精細にしたいときは 3 番目の数字で倍率指定: save fig.png 1200 1100 3（→ 3600×3300 px）。あるいは単純に大きい寸法を指定（save fig.png 2400 2200）。

map save は § 4.41、プレート境界は § 4.43、コンターは § 4.9、グラティキュールは § 4.8、震源カタログは § 4.10 を参照。

4.8. 緯度経度グラティキュール

map layer graticule on        # 緯度経度グリッド + 度ラベルを表示（既定 ON）
map layer graticule off       # 非表示

緯度経度グリッドを描画し、度ラベル（8°E 6°N 形式、白ハロー付き）を出す。map fit などでローカルにズームしたビューで現在地・座標を読み取るための基礎レイヤ。全 4 投影に対応:

  • Plate Carrée: ズームに応じて round number（1°/2°/5°/10°/30°…）間隔の直線グリッド。下辺に経度・左辺に緯度ラベル。
  • AEQD / Mollweide / Orthographic: 固定 30° のグローバルグリッド（曲線）。ラベルは各グリッド線の端付近に配置。Orthographic は手前半球のみ描画。

SVG 出力（map save）にもミラーされる（ラベルは Illustrator 互換のため度記号を省いた 8E 形式）。

4.9. 震央距離コンター

map contour 30    # 30° おきの同心円（30/60/90/120/150°）
map contour 10    # 10° おき
map contour off   # 非表示
map contour       # 現在の設定を表示

各震源を中心に、震央距離 (epicentral distance) Δ の 同心小円を <deg>° おきに描画。細い灰色破線 + 北側に距離ラベル。

ユース: - 「ある領域を調べたい時、どの Δ 窓に観測点を置くべきか」を視覚的に検討（例: PKP なら 144-180°、Sdiff なら 100-140°） - テレ地震解析の距離感覚を教える教材 - 震源-観測点配置の事前設計

投影ごとの見え方: - AEQD（震源中心）: 真円（中心が震源と一致する場合） - Plate Carrée / Mollweide: projection に応じて歪んだ環、antimeridian で自動分割 - Orthographic: 半球内のみ描画（裏側は自動カット）

複数震源がある場合は全震源について描画される。map save の SVG 出力にも反映。

4.10. 世界震源カタログ overlay

map layer quakes on           # 世界の震源を地図にプロット
map layer quakes off          # 非表示（既定）

USGS FDSN カタログ（M≥5.5・1950年〜現在・eventtype=earthquake、約 32,700 events）を地図に重ねる。

  • 色 = 深さ: 既定は 紫（浅発 0 km）→ 緑（中発 ~300 km）→ 赤（深発 ~700 km）の専用カラースケール。下辺左の凡例（earthquake depth (km)、0/70/300/700 km 目盛り = 浅発/中発/深発の分類境界）で色と深さの対応が読める。色方向は map style quake-color shallow-cool|shallow-warm で反転可（shallow-cool = 浅発が寒色＝既定、shallow-warm = 浅発が暖色）。Wadati-Benioff 帯（沈み込むスラブに沿う地震面）が深さ色で浮き上がる。
  • dot サイズ = マグニチュード: 既定で M5.5 ≈ 0.7 px 〜 M9 ≈ 4 px。map style quake-size <0.3–5> で全体倍率を調整（ERI 震源地図のようにマグニチュードの差を強調したいとき大きめに）。
  • 重なり順: map style quake-order auto|shallow-top|deep-top。auto（既定）は暖色端（＝注目色）を上に描く＝quake-color に追従（shallow-cool なら深発が上、shallow-warm なら浅発が上）。deep-top だと密な浅発帯の下にスラブ形状（深発）が見える。
  • データはビルド時に内蔵データとしてインライン埋め込み（5 整数/event に圧縮、約 470 KB）。ネットワーク不要。
  • 非地震イベント（核実験・採石発破）は eventtype=earthquake フィルタで除外済み。

教材用途: プレート境界（map layer plate）・プレート運動（map layer motion）と重ねると、地震分布・テクトニクス・スラブ形状が 1 画面で説明できる。時間・Mw・深さ・領域マスクは § 4.12 / -i、震源カタログ（USGS / 気象庁）の切替は § 4.14 を参照。

4.11. 核実験 overlay

map layer explosions on       # 地下核実験を表示
map layer explosions off      # 非表示（既定）

USGS カタログの地下核実験（1951-2017・約 1,000 回。ネバダ 677 / セミパラチンスク 146 / ノヴァヤゼムリャ / ロプノール / 北朝鮮 ほか）を地図に重ねる。

  • 爆発は等方震源 — あらゆる方向に圧縮を放射するので、発震機構（ビーチボール）は節面のない一様な圧縮 = 中実の円になる。断層地震の 4 象限ビーチボールとは見た目が全く違う。
  • オレンジの中実円（濃い縁）で描画、サイズは Mw 比例。
  • map layer quakes on ＋ map layer explosions on を同時表示すると、震源分布（点）と核実験（中実円）が並んで「地震 vs 爆発」を一目で見分けられる。地震学・CTBT 監視の教材として有効。（発震機構図での比較は § 4.39 の map event fm で仮想イベントに付与すれば可能）

4.12. タイムスライダー（時間窓）

map time 2011 2025                # 2011〜2025 年に絞る
map time 2011-03-11 2011-06-01    # 月日指定（2011 東北沖の余震系列）
map time 2011-03                  # 2011 年 3 月
map time 2011-03-11T14 2011-03-11T18  # 時刻指定（本震直後 4 時間）
map time off                      # 全期間に戻す
map time                          # 現在の窓を表示
map time 2016-04-14 2016-07-16    # 再生範囲を設定（熊本 2016 の余震系列）
map time play                     # その範囲内で窓をスライド＆ループ再生
map time play 3                   # 3 倍速
map time stop                     # アニメーション停止（Esc でも停止）
map timezone JST                  # 表示・入力タイムゾーンを JST に
map timezone UTC                  # UTC/GMT に戻す（既定）
map timezone +9                   # ±時間オフセットでも指定可

map layer quakes / explosions のいずれかが ON のとき、地図の下端にタイムスライダー（時間窓バー）が表示される。

  • カタログは 時刻分解能（1950-01-01 からの経過時間、時間単位）で時刻を保持。map time の from/to は YYYY / YYYY-MM / YYYY-MM-DD / YYYY-MM-DDThh で指定でき、余震系列（時間〜日スケール）も窓で切り出せる（例: 2011 東北沖 本震直後 4 時間 → 69 地震）。
  • タイムゾーン: イベントは内部的にすべて UTC で保持。map timezone は表示と map time の入力解釈を切り替えるだけ（既定 UTC／GMT、JST ＝ +9、±h で任意オフセット）。JST 設定時は map time 2011-03-11T14 を JST 14:00 として解釈・表示する。
  • スライダーのハンドル（両端の青いバー）をドラッグで窓の各端を、窓の内側をドラッグで窓全体をスライド。スライダー上の窓ラベルは現在の窓（窓幅が短いと YYYY-MM-DD HH:00、長いと YYYY-MM-DD）と現タイムゾーン名を示す。CLI map time とスライダーは同じ窓を共有。
  • 窓の外のイベントは非表示。「時間を動かして」地震活動の時空間発展を見る — 沈み込み帯の地震系列、本震-余震、核実験の年代変遷など。
  • map time play — 時刻送りアニメーション。map time <from> <to> で設定した窓を再生範囲とみなし、その中を狭い窓（範囲の約 1/10 幅）がスライドしてループする。先に map time で系列の期間を設定してから play すれば、一つの余震系列の時空間発展だけを繰り返し再生できる（範囲未設定なら 1950–2026 年を通しで再生）。map time play 3 で 3 倍速。map time stop・Esc・スライダーを掴むと停止。
  • カバー範囲は 1950-2026 年（震源・核実験は 1950 年代〜）。
  • SVG 出力（map save）にも時間窓が反映される（窓内のイベントだけが書き出される）。

4.13. カタログマスク（規模・深さ・領域）

map mag 7                       # Mw ≥ 7 だけ表示
map mag 6 7.5                   # Mw 6.0〜7.5 の範囲
map mag off                     # 規模マスク解除
map depth 0 70                  # 浅発（深さ 0〜70 km）だけ
map depth 300                   # 深さ 300 km 以深だけ
map depth off                   # 深さマスク解除
map region 128 148 30 46        # 経度 128〜148°E・緯度 30〜46°N の箱
map region 170 -170 -20 20      # W>E は日付変更線をまたぐ箱
map region draw                 # 地図上をドラッグして箱を選択
map region off                  # 領域マスク解除

タイムスライダー（§ 4.12）の時間軸に加え、震源・ビーチボール・核実験の 3 レイヤを規模・深さ・領域でも絞り込める（Step III のマスク群）。すべて掛け合わせ（AND）で効く。

  • map mag <min> [max] — Mw 範囲マスク。引数 1 個なら下限のみ（上限 10）。「大地震だけ」「中規模だけ」の抽出に。
  • map depth <min> [max] — 震源深さ（km）マスク。引数 1 個なら下限のみ（上限 800）。浅発/深発の対比、沈み込みスラブの可視化に。核実験は深さ情報を持たないためこのマスクの影響を受けない。
  • map region <W> <E> <S> <N> — 経度・緯度の箱で空間的に絞る。W>E は日付変更線をまたぐ箱として解釈。map region draw で地図上をドラッグして箱を直接選択（ドラッグ中は橙の枠でライブプレビュー、確定すると青の破線枠）。Esc で描画キャンセル。
  • 選択中の領域は地図上に青い破線の箱で表示され、SVG 出力（map save）にも書き出される。
  • マスクはどの投影法でも機能する（領域箱の枠は投影の歪みに沿って描かれる）。

4.14. 震源カタログの切替（USGS / 気象庁）

map catalog usgs                # 世界カタログ（USGS、既定）
map catalog jma                 # 気象庁カタログ（日本、デモ subset）
map catalog load                # 完全版 jma-japan.json をファイル選択で読込
map catalog                     # 現在のカタログを表示

map layer quakes の点が引くカタログを切り替える。USGS（全球・M≥5.5）は粗いので、日本の余震分布で断層面を見たい用途には密な気象庁一元化震源カタログ（M≥2）が向く。

  • USGS（既定）: 全球・M≥5.5・1950年〜（§ 4.10）。
  • JMA（デモ subset）: map catalog jma で、バンドル済みの代表4系列を表示 — 神戸1995・鳥取県西部2000・東北沖2011（3.11）・熊本2016 の余震系列（M≥2、計 57,086 events）。Sympa! Globe をそのまま開くだけで、密な余震雲＝断層面が見える。map catalog jma 切替時はビューポートを自動的に日本付近にズームする（全球表示だと余震が点にしか見えないため）。map catalog usgs に戻すと全球表示に戻る。
  • JMA（完全版）: map catalog load で jma-japan.json（日本全域・M≥2・1995-2023、757,155 events、約 18 MB）をファイル選択で読み込む。読み込むとデモ subset が完全版に差し替わる。完全版はこの教材版には同梱されない（jma-japan.json を別途入手して読み込む）。
  • 時間スライダー・タイムゾーン・各マスク（§ 4.12 / -i）はどちらのカタログにも効く。map catalog jma のあと map time で時刻窓を絞れば余震系列の時空間発展が追える。
  • 描画上限: 完全版を全域・マスク無しで表示すると重いので、可視 15 万点を超えると大きい順に間引く（規模しきい値方式）。map region / map time で絞れば全件描く。
  • データ出典: 気象庁一元化震源カタログ（気象庁が文部科学省と協力して処理した結果。気象庁・防災科学技術研究所 Hi-net・大学等の観測データを含む）。USGS は USGS Earthquake Catalog（FDSN）。

4.15. 地形（陸標高＋海底地形）

map layer bathymetry on           # 地形（陸の標高＋海底の水深）を表示
map layer bathymetry off          # 非表示（既定）

海岸線だけの地図に、陸の標高と海底の水深を色の濃淡で重ねる。山脈・大陸棚・海嶺・海溝・深海平原が地形として見え、プレート境界（map layer plate）と重ねるとプレートテクトニクスの教材になる。

  • 陸は段彩（緑＝低地 → 茶＝高地 → 淡灰＝高峰、8 段: 0 / 200 / 500 / 1000 / 2000 / 3000 / 4000 / 5000 m〜）。海は深度濃淡（薄青＝大陸棚 → 濃紺＝海溝、12 段: 0 / 200 / 1000 / … / 10000 m〜）。
  • データ: 海域は Natural Earth 10m 海底地形、陸域は ETOPO1 全球起伏モデル（NOAA）を 0.25° グリッドに統合してビルド時にインライン同梱（RLE 圧縮で約 600 KB）。
  • 凡例（左下）に陸標高バー（緑〜茶）と海底深度バー（青）の 2 本が出る。
  • 全投影法対応（各ピクセルを逆投影して標高グリッドを参照する方式 — ポリゴン塗りのはみ出しが起きない）。SVG 出力（map save）にも PNG ラスタとして埋め込まれる。
  • 出典: 海底地形 = Natural Earth、陸標高 = ETOPO1 Global Relief Model（NOAA NCEI）。

タイントのデータ切替（map tint）

map tint relief                   # 地形（陸標高＋海底地形、既定）
map tint age                      # 海洋底の年代（EarthByte）
map tint strain                   # ひずみ速度（GSRM v2.1）
map tint heat                     # 地殻熱流量（Lucazeau 2019）
map tint sediment                 # 海洋堆積層厚（GlobSed v3）
map tint magnetic                 # 地磁気異常（EMAG2v3）
map tint                          # 現在のデータセットを表示

map layer bathymetry のタイントレイヤは、複数の全球データセットから1つを選んで表示できる。map tint で切替（map layer bathymetry が OFF なら自動で ON）。

  • relief（既定）— 陸標高＋海底地形（§ 4.15 本文）。
  • age — 海洋底の年代。中央海嶺で 0、離れるほど古い → 海洋底拡大の対称縞が一目で読める。赤（若い・海嶺）→ 青紫（古い）の段彩。海洋のみ。出典: EarthByte 全球年代グリッド（2020、Müller et al. 系）。
  • strain — ひずみ速度（地殻変形速度、nanostrain/yr＝10⁻⁹/yr）。GSRM v2.1（Global Strain Rate Model、Kreemer et al. 2014）の測地ひずみ速度テンソルの第二不変量 √(ε̇₁²+ε̇₂²) を対数スケールで段彩（淡黄＝遅い → 濃赤＝速い、13段）。GSRM は変形帯（プレート境界付近）のみをモデル化し、剛体プレート内部はメッシュに含まれない → 内部は無着色のまま＝「地殻は狭い帯状の変形帯でのみ変形し、内部は剛体的に動く」というプレートテクトニクスの核心がそのまま見える。map layer plate・map layer quakes と重ねると、変形帯＝地震帯＝プレート境界の対応が読める。出典: GEM GSRM v2.1（Kreemer, Blewitt & Klein 2014, G-Cubed）。
  • heat — 地殻表面熱流量（mW/m²）。Lucazeau (2019) の予測熱流量を全球 0.5°グリッドで段彩（青＝低い → 赤＝高い、線形 35–150、13段）。中央海嶺・リフト・若い火山地域で高く（マントルからの熱供給）、古い海洋底・先カンブリア紀の安定大陸（クラトン）で低い。age（海洋底年代）と対で見ると「若い海洋底ほど熱い」海洋底冷却の関係が読める。出典: Lucazeau, F. (2019), Analysis and Mapping of an Updated Terrestrial Heat Flow Data Set, G-Cubed 20, 4001–4024。
  • sediment — 海洋堆積層の厚さ（m）。GlobSed v3（Straume et al. 2019）の全海洋堆積層厚を 5分グリッドから対数スケールで段彩（淡色＝薄い → 濃茶＝厚い、10–10000 m、13段）。海洋のみ（大陸は無着色）。中央海嶺は薄く（できたばかりの海洋地殻＝堆積物ほぼなし）、大陸縁辺の海底扇状地で極端に厚い（世界最大のベンガル海底扇状地、メキシコ湾、アマゾン扇状地など — 大陸から運ばれた砕屑物が厚く堆積）。age と対で見ると「古い海洋底ほど堆積物が厚い」関係が読める。出典: GlobSed v3（Straume et al. 2019, G-Cubed 20, 1756–1772、NCEI G01065）。
  • magnetic — 地殻の磁気異常（nT）。EMAG2v3（Earth Magnetic Anomaly Grid、Meyer, Saltus & Chulliat 2017）の上方接続（高度4 km）リソスフェア磁気異常を発散配色で段彩（紫＝負 ⇔ 淡色＝ほぼ0 ⇔ 橙＝正、±250 nT、13段）。Kursk 磁気異常・Bangui 異常など地殻の磁気構造の把握に。なお EMAG2 は全球コンパイルで外洋の磁気探査データが疎なため、海洋底拡大の鮮明な磁気縞（教科書的な平行縞）は出ない（まだら状の磁気異常を示す）— 海嶺対称の縞模様を見たいときは map tint age（海洋底年代＝磁気縞を同定・年代づけたもの）の方が明瞭。出典: EMAG2v3（Meyer et al. 2017、NCEI、2分グリッドの UpCont 版を 0.5°に集約）。

⚠ CRUST1.0 系の地殻データ（crust＝地殻厚、crustvp／crustvs＝地殻速度構造）と重力系データ（gravity＝フリーエア重力異常、bouguer＝ブーゲー異常）は、この教材版には同梱していない。

津波到達時間（map tsunami）

map tsunami 142.9 38.1            # 震源 (経度,緯度) からの津波到達時間
map tsunami play                  # 波面が広がるアニメーション（既定16秒）
map tsunami play 30               # 再生時間を秒で指定
map tsunami pause                 # 一時停止（再度 play で再開）
map tsunami stop                  # 停止して全到達時間マップに戻す
map tsunami movie                 # アニメーションを WebM 動画で書き出し
map tsunami movie 20              # 再生時間20秒で録画
map tsunami off                   # タイントを relief に戻す

津波は浅水波で、速さは c = √(g·d)（d＝水深）。map tsunami <経度> <緯度> は、その震源から波面が各海点に届くまでの時間を海底地形グリッド上の最短時間探索（ダイクストラ法）で自前計算し、タイントとして表示する（同心の色帯＝2時間ごとの波面）。陸地は障壁。外部データ不要 — Sympa!が地形を持っているので計算できる。例: map tsunami 142.9 38.1（2011 東北沖の震源）→ 環太平洋への津波到達時間。

アニメーション（時間発展）。 到達時間を計算したあと map tsunami play で、震源から波面が広がる様子をアニメーション表示できる。色の付いた領域が到達済み、明るい縁が現在の波面、左下の凡例に経過時刻（T+時:分）が出る。pause で一時停止（play で続きから再開）、stop で停止。map tsunami movie は同じアニメーションを WebM 動画ファイルに録画してダウンロードする（教材・発表用）。再生時間は play/movie の後ろに秒数で指定でき（既定 play=16秒・movie=14秒、2〜120秒）、震源から最遠点までの全到達時間がその秒数に線形に対応する。投影法・中心・ズームは再生中も自由に変えられる。

4.17. ホットスポット（mantle plumes）

map layer hotspots on             # ホットスポット／マントルプルームを表示
map layer hotspots off            # 非表示

世界のホットスポット約46（マントルプルームに由来する火成活動の中心）を、名称つきの暖色の円でプロットする。

  • 一次プルーム（深部マントル由来）を区別: Courtillot, Davaille, Besse & Stock (2003)「Three distinct types of hotspots in the Earth’s mantle」(EPSL 205) の判定基準（直線的な火山列・起点の洪水玄武岩・浮力フラックス・高 ³He/⁴He・直下の低速度異常）を満たす 7つの「一次（primary）」ホットスポット（Iceland, Hawaii, Easter, Louisville, Afar, Réunion, Tristan）にハロー（二重円）を付ける。これらはコア・マントル境界付近に根を持つ深部プルームの最有力候補。
  • それ以外のホットスポットは琥珀色の小さな円。火山レイヤ（map layer volcanoes）と重ねると、ホットスポット火山との対応が見える。
  • 各ホットスポットに名称ラベルが付く。混雑するときは map style hotspot-label off でラベルだけ消せる（マーカーは残る）。凡例は地図左上。

4.18. 大規模火成岩区（LIP／洪水玄武岩）

map layer lip on                  # LIP（洪水玄武岩・海台）を表示
map style lip-data johansson      # データ源: 海洋海台＋大陸/顕生代（既定, 連結つき）
map style lip-data ernst          # データ源: 大陸洪水玄武岩＋岩脈群/先カンブリア〜現在(~1272Ma)
map style lip-scope major         # 主要 LIP のみ（既定, ~27件・ホットスポット連結つき）
map style lip-scope all           # データセット全プロビンス（johansson=143/ernst=58）
map style lip-style outline         # 実分布の輪郭（半透明ポリゴン, 既定）
map style lip-style marker         # 円/菱形マーカー（色=年代/大きさ=面積/形=種別）
map style lip-style both            # マーカー＋輪郭
map style lip-link on|off          # LIP→ホットスポットの連結線（既定 on）
map style lip-label on             # 名前ラベル: on=既定(常に表示) / auto=マーカー時のみ / off=非表示
map demo floodbasalts             # 洪水玄武岩とホットスポットのデモ（地球儀／全球）

大規模火成岩区（Large Igneous Province; 大陸洪水玄武岩・海洋海台・珪長質 LIP）を、プルームの「頭」（短期間に巨大噴出した LIP）と「尾」（現在まで続くホットスポット軌跡）の関係として可視化する。

  • 2つのデータ源（map style lip-data）:
    • johansson（既定）— EarthByte LIPs v2（Johansson et al. 2018）。海洋海台（Ontong Java, Kerguelen 等）＋大陸 LIP、顕生代・現在位置。主要27件にホットスポット連結。lip-scope all で海山列・受動的縁辺火山等を含む全143プロビンス。
    • ernst — Ernst & Youbi (2017) / Ernst et al. (2021) の大陸洪水玄武岩＋岩脈群を、先カンブリア〜現在（最古 Mackenzie ~1272 Ma）まで収録（58件）。深部時間の LIP バーコード記録。海洋海台は含まない（性質上）。カラースケール上限は 1300 Ma に自動拡張。
  • 既定は実分布の輪郭（outline）＋名前ラベル。map style lip-style marker で記号表示、both で両方。
  • マーカーの3つの符号化（lip-style marker／both時）: 色＝噴出年代（明るい黄＝新しい → 暗い紫＝古い, viridis ランプ。上限はデータ源で変わる。凡例は左下のカラーバー）、大きさ＝面積（√面積に比例）、形＝種別（●大陸洪水玄武岩 CFB／◆海洋海台／▲珪長質 LIP）。
  • プルーム連結線: 主要 LIP から、文献で対応づけられる現在のホットスポットへ大圏（great circle）で線を引く。実線＝明瞭な軌跡で結ばれるもの（Deccan→Réunion, Paraná／Etendeka→Tristan, Columbia River→Yellowstone, Afro-Arabian/Ethiopia→Afar, Caribbean→Galápagos, Kerguelen 海台・Rajmahal→Kerguelen, Madagascar→Marion, NAIP→Iceland など）、点線＝対応が議論されているもの（Ontong Java／Manihiki／Hikurangi→Louisville, Emeishan→Hainan）。Siberian Traps・CAMP・Karoo・先カンブリア LIP 等は明瞭な現存ホットスポット軌跡を持たないため連結なし。
  • 輪郭モード（lip-style outline／both）は LIP の実分布域を半透明ポリゴンで描く（年代色）。地球儀・モルワイデ等どの投影でも日付変更線・地球縁を正しく処理する。輪郭モードで名前も出したいときは map style lip-label on。
  • 凡例（年代カラーバー＋種別記号）は地図の右下。段彩（map tint）のカラーバーは左下なので重ならない。
  • 出典: johansson = Johansson, Zahirovic & Müller (2018) GRL 45, 5380–5389（doi:10.1029/2017GL076691）。ernst = Ernst & Youbi (2017) Palaeogeogr. Palaeoclimatol. Palaeoecol. 478, 30–52 ＋ Ernst et al. (2021) AGU Geophys. Monogr. 255 Ch.1（輪郭は Park et al. 2020 経由）。年代・面積・重心・ポリゴンはデータセット値、LIP↔︎ホットスポット連結は Courtillot et al. (2003) / Ernst (2014) のキュレーション。refs コマンドでも表示。

4.19. ホットスポット軌跡（プルームの尾・海山列）

map layer hstracks on             # ホットスポット軌跡（年代進行する海山列）を表示
map layer hotspots on             # 現在のホットスポット（軌跡の起点）も一緒に
map layer lip on                  # LIP（軌跡の古い端＝プルームの頭）と対で

マントルプルームの「尾」＝現在のホットスポットから過去へ伸びる年代進行した海山列を描く（map layer lip の LIP＝プルームの「頭」と対）。

  • 計算方法: 固定ホットスポット仮定で、Matthews et al. (2016) のプレート回転モデルから自前計算。時刻 t にホットスポット直上にあったプレート上の点が現在どこにあるか＝海山列の軌跡。外部の年代データに依らず再現可能な自前計算。
  • 線の色＝年代（LIP レイヤと共通の viridis スケール）。なので軌跡の古い端は、対応する LIP の色と一致する（例: Réunion 軌跡の 66 Ma 端＝Deccan Traps の色で繋がる）。
  • 収録: Hawaii–Emperor（約47 Ma の折れ曲がり＝太平洋プレート運動方向の変化が見える）, Louisville, Réunion→Deccan, Tristan／Gough→Walvis→Etendeka, Kerguelen, Marion, Iceland, Yellowstone→Columbia River, Easter, Galápagos, Afar, Caroline, Tasmantid, Cobb, Samoa, Society, Marquesas, St. Helena, Trindade, Crozet（計21本）。
  • 海嶺をまたぐ軌跡（Réunion, Tristan 等）は時代ごとに所属プレートが変わるので、その都度プレートを判定して計算（板境界では軌跡を分割）。共役のもう一方の枝（例: Tristan の南米側 Rio Grande Rise→Paraná）は現在ホットスポットが乗る側のみ描画。
  • 注意（教材としての近似）＋ 約50 Ma より古い区間は点線: 固定ホットスポット仮定なので、実際の Ar-Ar 年代や海山位置とは厳密一致しない。約50 Ma より古い区間を点線にして「近似が弱い領域」を区別している。代表例がハワイ: プルーム自身が 80→47 Ma に約15°南へ移動した（Tarduno et al. 2003, Science）ため、固定と仮定した計算軌跡は天皇海山列で実際の海山より南にずれる（屈曲点で数度、80 Ma で約7°）。このずれ自体がホットスポットが完全には固定でない証拠で、固定モデル予測と実際の海山列の差を見る教材になる。若い側（ハワイ海山列, 0–47 Ma）はよく一致する。
  • map layer lip の連結線（LIP↔︎ホットスポットの直線大圏＝模式）に対し、本レイヤは実プレート運動に沿う曲線軌跡。両者を比べると「模式 vs 実際の道筋」が分かる。
  • 出典: Matthews, K. J., et al. (2016)「Global plate boundary evolution and kinematics since the late Paleozoic」Global Planet. Change 146, 226–250。refs コマンドでも表示。

4.20. 応力場（World Stress Map）

map layer stress on               # 地殻応力（SHmax 方位）を表示
map layer stress off              # 非表示

地殻の最大水平圧縮応力 SHmax の方位を、各サイトに短い線分（tick）で表示する。World Stress Map 2016（Heidbach et al.、GFZ）の品質 A–D データを 1°グリッドで間引いた約6,600サイト。

  • SHmax は軸（向きでなく方位）なので、線分はサイトを中心に矢印なしで描く。長さは一定。
  • 応力レジーム別に色分け: 赤＝正断層型（NF、伸張）、緑＝横ずれ型（SS）、青＝逆断層型（TF、圧縮）、灰＝不明（U）。同じ方位でも、地殻が伸びているか縮んでいるかが色でわかる。
  • 線分の方位は全投影で正しく描かれる（局所東西南北の座標基底を投影から数値的に求める — プレート運動矢印と同じ方式）。
  • map layer plate・map layer motion と重ねると、応力方位がプレート境界・プレート運動とどう関係するか（応力場＝プレート駆動力の指標）が読める。
  • 出典: World Stress Map Database Release 2016（Heidbach et al., GFZ、doi:10.5880/WSM.2016.001、品質ランク A–D）。

4.21. スロー地震（Slow Earthquake Database）

map sloweq load                   # スロー地震カタログ CSV を読み込む（ファイル選択）
map sloweq add                    # 別の CSV を追記（地域・種別を合成）
map sloweq clear                  # 読み込んだデータを破棄
map sloweq info                   # 件数・種別内訳・カタログ別の種別/位置/出典を表示
map layer sloweq on|off           # 表示の ON/OFF
map style sloweq-type tremor      # 微動だけ（lfe | vlf | sse | … / all で全部）
map style sloweq-size 1.5         # 点サイズ倍率（0.3〜5）

深部低周波微動・低周波地震（LFE）・超低周波地震（VLF）・スロースリップ（SSE）などのスロー地震を、種別で色分けした点でプロットする。通常の地震が固着域（地震発生層）で起きるのに対し、スロー地震は沈み込み境界の深部延長（down-dip 側）に並ぶ — map layer plate（プレート境界）・map layer quakes（通常の地震）と重ねると、「固着域の深部にスロー地震帯がある」というプレート境界すべりの全体像が 1 画面で見える。

  • 読み込み式（同梱なし）: スロー地震データは Slow Earthquake Database（Kano et al., 2018, SRL 89(4), 1566–1575; 「スロー地震学」プロジェクト、https://eqdb.sakura.ne.jp/ — アプリ内では refs sloweq）の利用規約（再配布禁止・非商用・各カタログの個別ポリシー遵守・出典明記）に従い、Sympa!には同梱していない。利用者が同サイトの Map+DL から必要なカタログ（地域・種別・期間を選択）を統一フォーマット CSV でダウンロードし、map sloweq load で読み込む。
  • 対応フォーマット: 同 DB の 3 つの出力フォーマット（year,…,lat,lon,dep,mag,catalog,ref,udate / モーメントテンソル付き / 断層パラメータ付き）をいずれも自動判別して読む。種別はカタログ名の接尾辞（NIED-Tremor・Aso2011-LFE・Asano2015-VLF・Nishimura2013-SSE 等）から自動判定。
  • 色分け: 微動＝橙 / LFE＝赤 / VLF＝紫 / SSE＝青 / repeater＝緑 / swarm＝青緑 / broadband＝茶 / その他＝灰。SSE（すべり面＝面的）はやや大きめ・最前面で描く。凡例は地図左上。
  • 合成: map sloweq add で複数ファイルを追記できる（例: 南海の微動 CSV ＋ Cascadia の微動 CSV を重ねて環太平洋のスロー地震を一望）。
  • フィルタ・時間: map style sloweq-type で 1 種だけに絞れる。時刻を持つイベントは下端のタイムスライダー（§ 4.12）・map time / map region / map mag / map depth のマスクにも従う。
  • 読み込み時の解説: 読み込み（map sloweq load）・レイヤ ON（map layer sloweq on）時に、カタログごとに「種別・件数・位置（中心座標）」を CLI に表示する。例: Wech2016-Tremor (tremor): 11,356 @ 148°W 62°N（＝アラスカの微動）/ Igarashi2020-Repeater (repeater): 35,222 @ 136°E 35°N（＝日本の相似地震）。これで「どのカタログがどこに・何の種別か」が一目で分かる（あるファイルで日本に微動が出ず別の点が出る、等の混乱を防ぐ）。
  • 出典明記: map sloweq info が全カタログの種別・位置に加え参照（ref 列）も一覧表示する。論文・発表で使う際は各カタログの designated reference を引用し、プロジェクトを謝辞に記すこと（DB の規約）。

4.22. 自前の観測点リスト（map sites）

map sites load [<名前>]           # 観測点リスト CSV を読み込む（名前=cwdから／省略=ファイル選択）
map sites add  [<名前>]           # 別の CSV を追記（複数地域・複数網を合成）
map sites clear                   # 読み込んだリストを破棄
map sites info                    # 件数・グループ別・ネットワーク別の内訳を表示
map layer sites on|off            # 表示の ON/OFF
map sites color group             # 色分け: group（既定）| network | uniform
map sites labels on               # 各点に code ラベルを付ける（既定 off）
map style sites-label-size 14     # 局名ラベルのフォント px（既定 9。論文図で拡大）

任意の観測点・地点リストを地図に点で重ねる汎用レイヤ。code,lat,lon を持つ CSV を読み込み、各点を色付きドットで描く。地震波形と関係ない地点群（地殻変動観測網・GNSS点・任意のサイト一覧）を載せたいとき用。FDSN レイヤ（§前出、IRIS/EarthScope の同梱地震計網）とは別物で、利用者が持ち込む任意の CSV を描く。

  • 読み込み式（同梱なし）: データは同梱せず、利用者が CSV を map sites load で読む。列はヘッダ名で対応付け（列順自由・余分な列は無視）。必須: code（または station/name）・lat・lon。任意: network（無ければ code の . より前を自動採用）・group（色分け用カテゴリ。任意の文字列）・label（値が truthy な行だけにラベルを付ける＝一部の局だけ局名を出したい時。列が無ければ全点ラベル）。
  • 色分け（map sites color）:
    • group（既定）— CSV の group 列で色分け。地殻変動 DB 用に both（歪+傾斜の両方を持つ点＝緑）/ strain（歪＝青）/ tilt（傾斜＝赤）/ aux（補助＝紫）に意味色を割当。未知の group 値はパレット色を順に割当。
    • network — ネットワークコードでパレット色を割当（多数網の分布を見るとき）。
    • uniform — 全点同色。
  • ラベル（map sites labels on|short|full|off）: 各点の下に観測点名を表示（既定 off）。on＝short＝ネットワーク接頭辞なしの局名（TU.KNK→KNK）、full＝フルコード（TU.KNK）。
  • 合成: map sites add で複数 CSV を追記できる（例: 東北の地殻変動点 ＋ 別地域の点を重ねる）。
  • 読み込み時の挙動: ブラウザでは読み込み後にビューを自動で観測点範囲にフィット。CLI に「件数・group 別内訳・network 別内訳」を表示。
  • SVG 出力: 点・凡例・ラベルとも SVG 出力に反映（Illustrator 互換、map save *.svg）。
  • ヘッドレス（render.js）: map sites load <path> でファイルパスを直接指定して読める（スクリプトから観測点マップを自動生成可）。
  • 地域へ枠取り: map fit <W> <E> <S> <N>（例 map fit 136 145 35 42 で東北）、map fit japan、map fit sites（読み込んだ観測点の範囲）でビューを一発でズーム＋センタリングできる。
  • 海岸線・標高/海底を精細に: 地域ズームでデフォルト（NE 50m 海岸線・0.25° relief）が粗いときは map detail on 一発で hi-res（NE 10m 海岸線 ＋ 0.05° relief、約5倍精細）に切替（ビルドに同梱＝ファイル選択不要）。map detail off で戻す。外部の独自パックを使うときだけ map detail load <file>。
  • さらに細かい自前海岸線: 内蔵は NE 10m（~280 m 量子化）が最細。GMT/GSHHG 級（例: GSHHG full-res から作った三陸リアス海岸）が要るときは map coast load <file> で TopoJSON {transform, arcs} の海岸線を読み込み、coastline レイヤをそれに差し替える（map coast off で内蔵へ復帰）。論文グレードの地域図向け。render.js でも map coast load <path> 可。
  • データ読み込みコマンドの再現性（map … load <名前>）: map scalar/coast/coast land/sites load は、名前を付けると cd <フォルダ> 済みの作業フォルダから直接読み込む（r と同じ仕組み）。名前を省くとファイルピッカー。これにより作図レシピを GUI の CLI でもそのまま再生できる（例: cd ~/…/keeper_figs → map scalar load <name>.csv → map coast load … → …）。ヘッドレスの render.js ではパス（絶対/相対）指定で同じレシピが動く。つまり同じコマンド列が GUI でも headless でも再現可能。

4.23. 有限断層すべり分布（map slip）

map slip load                     # 有限断層 GeoJSON を読み込む（ファイル選択）
map slip clear                    # 破棄
map slip info                     # subfault 数・最大すべりを表示
map layer slip on|off             # 表示の ON/OFF
map slip min 5                    # 5 m 未満のすべりを隠す（主アスペリティだけ表示）

大地震の断層面の「すべり量分布」を、塗り分けたサブ断層パッチで地図に重ねる。「どこでどれだけ滑ったか」（例: 2011 東北沖 M9 は海溝寄りの浅部で最大 ~55 m）を観測点・震源と一緒に見せられる。

  • 読み込み式（同梱なし）: USGS の震源ページ → Finite Fault プロダクトの FFM.geojson（サブ断層ポリゴン＋slip[m] 属性）をダウンロードして map slip load で読む。USGS は public domain（出典明記推奨）。汎用パーサなので、slip プロパティを持つポリゴンの GeoJSON FeatureCollection なら他イベントでも動く。
  • 配色: すべり量の逐次配色（淡黄→赤、凡例は左上のカラーバー[m]）。map slip min <m> 未満のサブ断層は描かない＝主すべり域だけを浮かせる（ライブでしきい値調整）。
  • 重ね合わせ: 観測点（map sites）・震源（map event add）・余震（map layer quakes）と重ねると、すべり域と地表観測網・余震分布の位置関係が 1 枚で分かる。
  • SVG 出力: パッチ・カラーバーとも SVG に反映。render.js は map slip load <path> でヘッドレス可。

4.24. 汎用スカラー場（map scalar — lat,lon,value グリッドのヒートマップ）

map scalar load [<名前>]          # (lat,lon,value) の CSV を読む（名前=cwdから／省略=ファイル選択）
map scalar clear                  # 破棄
map scalar info                   # セル数・値列名・値域を表示
map layer scalar on|off           # 表示の ON/OFF
map scalar cbar "<ラベル>"         # カラーバーのタイトル（未設定なら値列名）
map scalar col <列名>             # 値列を明示指定（次の load から有効）
map style scalar-color viridis|magma|plasma|gray|ocean   # カラーマップ（既定 viridis。ocean=海洋深度用 淡青→濃紺）
map style scalar-rev on|off       # 反転（on=低値を明色。検出限界マップ向き）
map style scalar-alpha 0.1-1      # 不透明度（既定 1）

任意の (lat, lon, value) グリッドを、値で色塗りしたセル（矩形タイル）＋カラーバーとして地図に重ねる汎用ヒートマップ。列はヘッダ名で対応付け（列順自由）。必須: lat・lon。値列は lat/lon 以外の最初の数値列を自動採用（map scalar col <列名> で明示指定可）。セル幅はグリッド間隔から自動推定するので、規則格子なら隙間なく敷き詰まる。

  • 読み込み式（同梱なし）: 利用者が CSV を map scalar load で読む。例＝crust-db ひずみ計網の検出限界マップ（Slab2 界面上の各格子点で「網が検出できる最小 Mw」= net_Mw。小さいほど高感度、大きいほど死角）。
  • 配色: 知覚均等な連続カラーマップ（scalar-color）。scalar-rev on で反転すると低値が明色になり、検出限界マップの慣例（小さい Mw＝高感度＝「良い」を明色で強調）に合う。透過で濃淡を作らない作図規律に従い、既定の不透明度は 1。
  • カラーバー: 左上にタイトル（scalar cbar）＋下端に最小値/中央値/最大値。
  • 重ね順: セルは海岸線の下に描かれ、海岸線・観測点（map sites）・震源（map event add）がその上に乗る。観測点・震源・map style chrome off（ヘッダ status 行を隠す）と組み合わせると論文グレードの keeper 図になる。
  • SVG 出力: セル・カラーバーとも SVG に反映。render.js は map scalar load <path> でヘッドレス可（スクリプトから自動生成可）。

4.25. 沈み込みスラブ（Slab2 等深線）

map layer slab on                 # Slab2 スラブ上面の等深線を表示
map layer slab off                # 非表示
map style slab-labels off         # 深さラベル（km）を隠す（線だけ）

沈み込むスラブ（海洋プレート）上面の深さ等値線を地図に描く。Slab2（Hayes et al. 2018）の全 27 沈み込み帯（千島-日本・伊豆小笠原・南海/琉球・アンデス・カスケード・スマトラ・トンガ-ケルマデック ほか）の深さモデルから、20 km 間隔・0〜700 km の等深線をプロットする。

  • 深さで色分け（震源レイヤ・スロー地震レイヤと共通の 0〜700 km 深さカラーランプ）。だから 100 km の等深線と深さ 100 km の地震が同じ色になり、「震源やスロー地震がスラブ面のどの深さに乗っているか」が色で直読できる。map style quake-color shallow-warm で配色の向きも揃って反転する。
  • map layer quakes（通常の地震）と重ねる → 和達-ベニオフ面（沈み込むスラブ内の深発地震面）が等深線とぴったり重なるのが見える。map layer sloweq（スロー地震）と重ねる → 微動・SSE がスラブ上面の特定の深さ帯（固着域の深部延長）に並ぶのが見える。
  • 100 km ごとの等深線に深さラベル（km）が付く（map style slab-labels off で線だけに）。
  • 出典: Slab2 — Hayes, Moore, Portner, Hearne, Flamme, Furtney & Smoczyk (2018)「Slab2, a comprehensive subduction zone geometry model」Science 362, 58–61, doi:10.1126/science.aat4723（USGS、パブリックドメイン）。アプリ内では refs slab でこのリンクに飛べる。

4.27. GNSS 地表速度場（NGL MIDAS）

map layer gnss on                 # GNSS 観測速度ベクトルを表示
map style gnss-scale 2            # 矢印の長さ倍率（0.2〜5）
map style gnss-min 10             # 10 mm/yr 未満の点を間引く
map style gnss-color teal         # 矢印の色

世界約 20,000 観測点の GNSS 年間変位ベクトル（mm/yr）を矢印で描く。Nevada Geodetic Laboratory の MIDAS 解（IGS14 ＝ほぼ no-net-rotation の全球フレーム）。

  • 観測されたプレート運動: IGS14 はほぼ no-net-rotation なので、矢印は絶対的なプレート運動を示す（太平洋プレート ~7 cm/yr 北西、オーストラリア ~7 cm/yr 北北東、安定大陸内部は遅い）。map layer motion（MORVEL/GSRM モデル）の観測版にあたる — 両者を重ねるとモデルと実測の一致が見える。
  • 沈み込み帯の固着: map layer plate・map layer slab・map layer sloweq と重ねると、海溝に向かう弾性的なひずみの蓄積（プレート間固着）と、その深部延長で起こるスロー地震の関係が読める。

4.29. マントル地震波トモグラフィ（深さスライス）

map tomo                          # マントル Vs トモグラフィを表示（既定 S362ANI ~600km）
map tomo 2800                     # 深さ 2800km のスライス（最寄りにスナップ）
map tomo up                       # 一つ浅いスライス / down で深い
map tomo model s40rts             # モデル切替（下記8種。S40RTSは球面調和で滑らか）
map tomo models                   # モデル一覧
map tomo contour 0.5              # ±0.5% 間隔の等値線を重ねる（off で消す）
map tomo section 165 40 110 40    # 大円に沿った深さ断面図（lon1 lat1 lon2 lat2）
map tomo smooth off               # SH系モデルを生グリッド表示に（on で球面調和のなめらか表示に戻す）
map tomo vote 1                   # モデル合意度マップ（各モデルを深さ平均で規格化, 閾値1σ, off で解除）
map tomo vote fast                # 「速い」票数マップ(0〜N) ／ slow=「遅い」票数 ／ net=純票(既定)
map tomo compare s40rts s362ani   # 2モデルを同じ大円で上下に並べた断面比較（off で解除）
map tomo split s40rts s362ani     # 2枚の世界地図を左右に並置し左=A/右=B を同深さ・同範囲で比較（off で解除）
map split age relief              # ★汎用版★ 任意の2タイントを左右に並置（プレート年代 vs 標高 等。off で解除）
map tomo wedge on                 # 断面を扇形（地球を放射状に切った形）に（off で矩形）
map tomo section pick             # 地図を2点クリックして大円断面を切る
map tomo section ray P S PcP      # 断面に anisotime の波線を重ねる（PREM、断面のΔで計算）
map tomo section ray P S PcP depth 600   # 深発震源（源 600km）の波線。既定は地表源（depth 0）
map tomo section ray P S PcP model ak135 depth 600   # モデルと震源深さを同時指定（model/depth は順不同）
map tomo section ray PKP PKIKP    # コア相を指定すると断面が自動で地球中心まで延長される
map tomo section whole on         # 断面を全地球深さ（中心 6371km）に延長（off でマントルのみ）
map tomo section blank on         # 白断面図（トモ tint なしの素の地球。波線図向け、off で戻す）
map tomo section arc 360          # パネルの弧長を指定（180超で地球を一周＝円表示。off で震央距離に戻す）
map tomo section from 140 35 az 90 arc 360 recv 60   # 始点+方位+弧長+受信距離で断面を切る
map tomo off                      # タイントを relief に戻す
map tint tomo                     # 同上（tint データセットとしても選べる）

マントルの3次元構造を深さスライスで見せる。複数の全マントル地震波速度モデル（既定 S362ANI）の速度異常を深さスライスでタイント表示する。

  • 配色（SubMachine 風の発散カラーマップ）: 赤＝遅い（高温・上昇流。プルームや最下部マントルの LLSVP）、青＝速い（低温。沈み込んだスラブ）。中央＝平均。スケールはモデルにより ±2%（S波）/ ±1%（P波）。震源・スロー地震・スラブ等深線の深さ色とは別系統。

  • 深さスライス: map tomo <km> で最寄りの層にスナップ（モデルにより ~25〜40層、25〜2890km）。map tomo up/down で隣の層へ。

  • コンター map tomo contour <%>: 指定間隔（±%, ±2%, …）の dVs/dVp 等値線を細線で重ねる。off で消す。構造の境界（スラブの縁・LLSVP の輪郭）が読みやすくなる。地図・断面の両方に効く。

  • 深さ断面図 map tomo section <lon1> <lat1> <lon2> <lat2>: 指定した2点を結ぶ大円に沿ってマントルを縦に切った断面を表示する（横軸＝大円沿いの距離°、縦軸＝深さ 0〜2890km、色＝速度異常、410/660/CMB を破線表示）。JAMSTEC CSMAP や SubMachine の cross-section に相当。表示中に map tomo model <name> でモデル、map tomo contour <%> で等値線を切替えると断面に反映。map または Esc で地図に戻る（Esc は一度で section pick モードに戻って切り直せ、もう一度で地図）、map tomo section off で解除。例: map tomo model s40rts; map tomo section 165 40 110 40 で太平洋スラブの構造が見える。save section.svg/.png でベクトル/画像出力も可。

  • モデル切替 map tomo model <name>（深さは保持して各モデルの最寄り層に再スナップ）— 7モデルを同梱:

    • 滑らか系（球面調和でなめらか）: s362ani S362ANI（Kustowski et al. 2008、既定、deg 24）／seisglob2 SEISGLOB2（Durand et al. 2017、deg 40）／sglobe-rani SGLOBE-rani（Chang et al. 2015、放射異方性、deg 40）／saw642an SAW642AN（Mégnin & Romanowicz 2000、古典波形、deg 32）／s40rts S40RTS（Ritsema et al. 2011, GJI 184:1223、deg 40）。これらは球面調和係数で同梱し実行時に滑らかな連続場を合成する（グリッド由来のカクつきが無く等値線がなめらか＝SubMachine と同方式）。各 degree はモデルの素性に合わせて選択（往復誤差は場の 1% 未満）。係数なのでデータも軽量。
    • グリッド系（鋭い構造を保持）: gypsum GyPSuM-S（Simmons et al. 2010、地震波+ジオダイナミクス）／semucb SEMUCB-WM1（French & Romanowicz 2014、スペクトル要素波形）。
    • モデルを切り替えると、よく合う特徴（LLSVP の位置など）と、解像度・振幅の違いが比較できる。

    • ⚠ P波モデル GAP-P4（深尾・大林; Obayashi/Fukao 2013）は JAMSTEC GODAC の利用規約（再配布制限）により同梱していない。

  • なめらか/生グリッド切替 map tomo smooth on|off（SH 系モデルのみ）: on（既定）= 球面調和係数から合成したなめらかな連続場、off = 同梱の生グリッド（バンド量子化・2°）をそのまま表示。両者を切り替えることで、SH フィットがどれだけ忠実か（見た目の精度）を自分の目で確認できる。グリッド系モデル（GAP-P4 等）では効果なし。地図・断面の両方に効く。

  • モデル合意度マップ（votemap） map tomo vote <σ>（SubMachine 流、Hosseini et al. 2018, G-Cubed）: 現在の深さで各 S波モデルを規格化（その深さの cos緯度重み平均を引き、自身の標準偏差σで割る）し、各点を「速い（+1, 偏差>+t·σ）/遅い（−1, <−t·σ）/中立（0）」と投票して純票を表示する。+N（=全モデル）速い＝濃青／0＝拮抗・不一致＝白／−N 遅い＝濃赤（N＝投票モデル数。途中の値は部分合意）。各モデルを自身のσで規格化するので振幅較正・平均オフセットの差に依存しない——絶対値が信用できない差分マップの問題を回避するロバストな比較。しきい値は σ 単位で可変（既定 1σ＝SubMachine の “std” 指標相当、map tomo vote 0.5 で緩める）。P波の GAP-P4 は除外。off で解除。例: 2800km で太平洋・アフリカの LLSVP は強い遅い合意（濃赤）、楯状地直下は速い合意（濃青）、中マントルや遷移層滞留スラブの一部はモデルが割れて淡色＝強い構造ほど一致し、議論のある構造ほど淡くなる。

    • fast/slow/net の3表示（SubMachine 流）: map tomo vote fast =「速い」と投票したモデル数のマップ（0〜N, 白→青）、slow =「遅い」票数（0〜N, 白→赤）、net（既定）= 速い票−遅い票の純票（発散）。map tomo vote fast 1.5 のようにモードとしきい値を同時指定も可。スラブ（速い票数）やプルーム（遅い票数）を個別に追えるのが SubMachine と同じ使い方。

  • 扇形（wedge）断面 map tomo wedge on|off: 深さ断面を地球を放射状に切った扇形（半径＝地球半径−深さ、角度＝大円距離）で描く。深部ほど横幅が縮む真の断面形状。単一断面・並置比較の両方に効く。既定は扇形で、off で矩形に切替える。

  • 地図上で大円を指定 → 大円断面 map tomo section pick: コマンドで4座標を打つ代わりに、地図上で大円を designate して断面を切る。①始点をクリック → ②マウスで終点を合わせる（大円とその大円距離 Δ が地図上にライブ表示される。Δ は 5°刻みにスナップするので断面どうしを同じ距離で揃えて比較しやすい。Shift を押すと 1°刻み）→ クリックで終点を確定（大円が地図に残る）→ Enter で断面図を表示。クリック中の Esc で中止。断面を切った後も Esc で section pick に戻ってやり直せる（もう一度 Esc で地図へ）。

  • 断面に地震波線（ray path）を重ねる map tomo section ray <phase…> [model <名>] [depth <km>]: 指定フェーズ（P S PcP ScS PKP …）の波線を anisotime で計算し、断面の (距離°×深さ) 平面にそのまま重ねて描く。既定は断面の端から端まで（断面の大円距離 Δ）を、地表の震源-観測点として計算する（例: 50° の断面なら Δ=50° で到達する波線）。モデルは既定 PREM、model ak135 等で変更可。深発震源は depth 600 のように指定。フェーズごとに色分け＋凡例、断面の Δ に届かないフェーズはスキップ（CLI に表示）。三重点（410/660 が作る走時の折り返し）域では同じ Δ に複数の波線が出る（全分岐を描く）。map tomo section ray off で消去。トモグラフィ構造（スラブ・プルーム）の上に波線を重ねて「その相がどこを通るか」を見るのに使う。

  • コア相（core phase）と全地球深さ断面: PKP / PKIKP / PKiKP / SKS / SKKS などコアを通る相を section ray で指定すると、断面が自動で地球中心（6371km）まで延長され、外核で turning する PKP、内核を貫く PKIKP、ICB（5150km）で反射する PKiKP などが最後まで描かれる。CMB（2890km）に加え ICB を破線で表示。コア相を使わず手動で全地球にしたいときは map tomo section whole on（off でマントルのみ 0〜2890km に戻す）。マントルのトモ tint は外側のリングに残り、コア部分は tint なし（トモモデルがマントルのみのため）。

  • 白断面図（blank section） map tomo section blank on|off: トモグラフィの色を載せない素の地球の断面。波線図だけをすっきり見せたいとき（教材・波線幾何の説明）に使う。タイトルとカラーバーは省略され、不連続（410/660/CMB/ICB）・深さ目盛・波線はそのまま。コア相と組み合わせれば、走時表に頼らず PKP/PKIKP/PKiKP の経路を一目で示せる。例: map tomo section blank on; map tomo section ray PKP PKIKP; map tomo section 0 0 150 0。

  • 断面の弧を広げる／地球を一周（360°） map tomo section arc <deg>（1〜360°、off で解除）: 断面パネルの横の弧長を震央距離と独立に指定する。180°を超えると断面を地球まるごとの円として描く（震源を頂点＝12時、距離は時計回り、中心 6371km まで＝自動で全地球深さ。マントルのトモ tint は外側のリングに残りコアは無色）。360° は教科書的な全地球波線図になり、コア相（PKP/PKIKP/PKiKP）が震源を頂点に左右へ拡がる。off で受信点までの通常断面に戻す。扇形（wedge）では円、矩形（wedge off）では 0〜360° に展開した縦断面。

  • 方位＋弧長で直接指定 map tomo section from <lon> <lat> az <deg> [arc <deg>] [recv <deg>]: 2点を打つ代わりに、始点・初期方位・パネル弧長・受信点距離で断面を切る。az＝大円の向き、arc＝表示する弧長（既定180°）、recv＝波線を計算する震央距離（既定 = min(180, arc)）。例 map tomo section from 140 35 az 90 arc 360 recv 60 ＝ 始点(140,35)から方位90°の大円を一周表示し、60°地点を観測点として波線を引く。2点指定（≤180°）では届かない弧をワンコマンドで作れる。

  • 反対回り（mirror）の波線: 一周（360°）表示では、震源から逆方位へ出た波線も自動で描かれる（前方＝時計回りで受信点へ、後方＝反時計回りで反対側の同距離点へ）。左右対称の全地球レイダイアグラムになる。部分弧（270°等）では mirror は描かず前方の波線のみ。

  • 地図の左右並置（2枚並べ） map tomo split <モデルA> <モデルB>: 完結した世界地図を2枚、左＝モデルA・右＝モデルBとして左右に並べる（中央に区切り線＋モデル名）。両方が同じ深さ・投影・表示範囲なので、同じ地域を2モデルで直接見比べられる。map tomo <km> で両側の深さを同時に変更、パン/ズームも同期。off で解除。（※閲覧向け。地図 tint なので SVG 出力は未対応・ブラウザ表示のみ。クリック配置等の細かい操作は split off で。） > ⚠ 日本シームレス地質図 V2（map tint geology / map layer geology）と日本の地体構造区分（terrane subdivision）のデータは同梱していない。地質区分を見たい場合は § 4.46（全球地質プロビンス, Hasterok et al. 2022）を使うこと。

  • ★汎用版★ 異なる物理量の左右並置 map split <A> <B>: 上記 map tomo split の一般化。異なる2つのタイント（物理量）を左右に並べ、同じ視点・同じ投影・同じ表示範囲で世界地球を回しながら見比べられる（地球内部・表層の物理量を直接対比する教材として有効）。A/B は relief（標高）／age（プレート年代）／strain（ひずみ速度）／heat（熱流量）／sediment（堆積層厚）／magnetic（磁気異常）／tomo（マントルトモ・現在モデル）、またはトモモデルキー（s40rts・s362ani 等）（⚠ crust／crustvp／crustvs（CRUST1.0 系地殻データ）・gravity／bouguer（重力系データ）はこの教材版に同梱していないため選択不可）。例: map split age relief（海洋プレート年代と海底地形の対応＝ridge から離れるほど深くなる＆age が古くなる）、map split heat sediment（熱流量と堆積層厚＝中央海嶺で高熱流・薄い堆積／古い海洋底で低熱流・厚い堆積、age と対で見ると海洋底冷却の関係が読める）、map split strain relief（変形帯と地形の対応＝プレート境界の変形帯が海嶺・海溝の地形と重なる）。パン・ズーム同期、map split off で解除。map proj ortho でぐるぐる回せる正射投影にして並べると教育効果大。SVG 出力は単一タイントのみ＝ブラウザ表示専用。

  • モデル並置（断面比較） map tomo compare <モデルA> <モデルB>: 先に map tomo section で大円を指定し、2モデルの深さ断面を同じ大円に沿って上下に並べて表示する（上=A・下=B）。map tomo contour <%> は両断面に効く。off で解除。振幅較正の違いが気になる差分マップの代わりに、同じ profile を並べて直接見比べるのが目的。例: map tomo section 165 40 110 40; map tomo compare s40rts s362ani。（地図の左右並置は別途。）

  • 教材的な見どころ: 2800km（最下部マントル）= 太平洋・アフリカ直下の2大 LLSVP（巨大低速度域＝スーパープルーム） が赤、周囲の沈み込んだスラブの墓場が青。600〜1000km（遷移層〜下部マントル上部）= 沈み込み帯の下に滞留・沈降するスラブが青（特に P波の GAP-P4 でスラブが鮮明）。map layer plate・map layer slab・map layer quakes と重ねると、表層のプレート運動と深部構造の対応が読める。

  • 全投影対応（逆投影サンプリング）。map detail の高解像度地形とは独立。SEMUCB-WM1 は絶対 Vs(km/s) で配布のため各深さの横方向平均からの % 異常に換算。SH 系は元のグリッドから自前で球面調和に再フィットしたもの。

  • 出典: S系6モデルは IRIS/EarthScope EMC、GAP-P4 は JAMSTEC（⚠GAP-P4 は JAMSTEC GODAC のデータ利用規約に従うこと。配布時は要確認）。各原論文は本文中・refs tomo。

  • 出典: 各モデルの原論文（上記）＋ IRIS/EarthScope EMC。アプリ内では refs tomo、レイヤ表示時に現モデルの出典も表示。

4.30. 端点と楕円体補正

入力された (lat, lon) は state.ellipsoid.f に従って geodetic → geocentric 変換が一度だけ行われ、その値で球面大円を計算 + 投影する（computeBAZ と同じ規約）。これにより:

  • 観測点マーカーの位置と dist / baz の出力が機械精度で一致
  • 大円弧の端点が観測点マーカーにピタリ重なる

ellipsoid sphere（f=0）にすると補正が無効化され、純粋球面モードになる。

4.31. シンボル・色

  • 震源 ★: 赤系（Tableau パレット）。KEVNM 値をラベルとして上に表示
  • 観測点 ▽: トレース色を継承（recsec / overlay と整合）。KSTNM.KNETWK をラベルとして下に表示
  • 大円弧: 黒系。plate-carree で経度 180° を跨ぐ場合は自動分割

複数の波形ファイルが同じ震源座標（±0.01°、深さ ±1km 以内）を持つ場合は1つの ★ に集約される。観測点も座標一致（±0.001°）で集約。

多イベント/多観測点の色分け:

map style event-color palette または map style station-color palette で複数の震源 / 観測点を Tableau-10 カラーパレットで色分けできる。

色分けが ON のとき、地図左上（ヘッダー直下）に 凡例パネルが自動表示される（★ または ▽ のミニマーカー + ラベル）。1 クラスあたり最大 12 行で、それ以上は +N more と省略。map style legend off で凡例も含めて全部隠せる。

map 入力時に「イベント 2 件以上 + 色分け OFF」状態を検出すると、map style event-color palette を提案する hint を CLI に 1 度だけ出す。

4.32. 状態の集約タイミング

map を入力したとき、および map モードでの再描画時に state.files から自動集約される。rh <ファイル名> などで新規ファイルを追加した後は、再度 map を打てば最新の状態に更新される（描画はそのまま継続）。

4.33. マーカーの色と形

map ray <phase> でプロットされる ◆ ◇ マーカーは:

  • 色 = 深さ（右下のカラーバーと一致）。表面→青、上部マントル底→青緑、CMB→橙、ICB→赤、中心→紫
  • 形 = タイプ
    • ◆ turning point: マントル内で ray が最深点になる地点（direct P/S）
    • ◇ reflection: 境界面での反射（CMB の PcP/ScS、ICB の PKiKP、surface の PP/SS）

つまり ◆ で橙色なら「マントル下部で turning した深い P/S」、◇ で橙色なら「CMB 反射」と読める。形＋色で phase の挙動が一目で分かる。

4.35. 波線マーカー

theory <model> で走時カタログを構築済みの状態で map ray <phase> を打つと、各震源-観測点ペアの大円弧上に turning / reflection マーカーが表示される。

theory PREM             # まず走時カタログを構築
map                     # 地図モードへ
map ray P               # 直達 P の turning point マーカー（大円中点）
map ray PcP ScS         # CMB 反射マーカー（複数 phase 同時可）
map ray                 # 現在の ray 一覧
map ray del PcP         # ScS だけ残す
map ray off             # 全消去

サポート phase:

カテゴリ phase マーカー 色
直達マントル（turning） P, S ◆ 塗り 琥珀 (amber)
マントル底回折 Pd, Sd ◆ 塗り 琥珀
CMB 反射 PcP, ScS ◇ 中抜き 赤
ICB 反射 PKiKP ◇ 中抜き 紫
表面反射 PP, SS ◇ 中抜き 青
深さ phase（surface bounce） pP, sS, sP, pS ◇ 中抜き 青
コア phase（複数マーカー） PKP, SKS ◇◆◇（3個） 赤・橙・赤
コア phase（複数マーカー） PKIKP ◇◇◆◇◇（5個） 赤・紫・橙・紫・赤
コア phase（複数マーカー） SKKS ◇◆◇◆◇（5個） 赤・橙・赤・橙・赤

マーカー下に <phase> <depth>km ラベル（例: PcP 2891 km、P 660 km）。深さは catalog の ray parameter を findTurningR で逆引きして算出。深さ phase の surface bounce は surface ラベル。コア phase は同じ phase 名のマーカーが複数並ぶ（CMB / ICB / turning を Δ 順に）。

マーカー位置の厳密性: - P, S, PcP, ScS, PKiKP, PP, SS: 震源深さによる非対称（down-leg Δ < up-leg Δ）を bullenIntegral で厳密に計算し、turning / reflection 位置を正しい角度距離（震源寄り）に配置する。EVDP=0 のとき Δ/2 と一致 - pP, sS, sP, pS: 震源から地表反射点までの up-leg を ppUp の bullenIntegral で厳密計算。深発（例: d=579 km, Δ=60°）で bounce が event 側に 4° 程度まで寄る。EVDP=0 では catalog 自体が depth phase を生成しないので marker は出ない - PKP, SKS: 4 sub-leg（mantle down + OC K down/up + mantle up）。マーカーは CMB 入/OC turning/CMB 出の 3 点を Bullen 積分で厳密配置 - PKIKP: 6 sub-leg（mantle + K + I turning + K + mantle）。マーカーは CMB 入/ICB 入/IC turning/ICB 出/CMB 出の 5 点 - SKKS: 6 sub-leg（mantle SV down + K turn #1 + CMB 下反射 + K turn #2 + mantle SV up）。マーカーは CMB 入/K turn #1/CMB 下反射/K turn #2/CMB 出の 5 点。中央 CMB 反射が SKS との見分け方 - Pd, Sd: CMB 上の長い回折弧に単一マーカーは意味が薄いので Δ/2 の placeholder

4.36. 深さ帯ハイライト

map layer raypath on（既定 ON）+ map ray <phase> で、大円弧を ray の深さに応じて色分けして描画する。

  • 色マッピング: 表面（青）→ 上部マントル底（青緑）→ CMB（橙）→ ICB（赤）→ 中心（紫）の HSL グラデーション。境界（660, 2891, 5150 km）でアンカー
  • 大円弧上に N サンプル（coarse=16, accurate=32）の (θ, depth) を bullenIntegral で生成し、隣接サンプル間を中点深さの色で線分描画
  • 右下に 深さ凡例カラーバー（0/660/2891/5150/6371 km の tick 付き）
map ray P                # → 大円弧が深さ色で塗られる（表面から ~700km 付近を山なりに往復）
map ray PcP              # → CMB 反射、両 leg がマントル色、中点で橙
map ray PKiKP            # → マントル + 外核 + 反射 + 外核 + マントル の色変化
map layer raypath off    # → 黒い大円弧に戻す（マーカーは残る）

サポート phase: P, S, PcP, ScS, PKiKP, PP, SS, pP, sS, sP, pS, PKP, PKIKP, SKS, SKKS（pathAt 実装済）。深さ phase は 3 sub-leg、コア phase は 4-6 sub-leg で着色が連続的に変化（mantle 緑 → OC 橙 → IC 紫 等）。Pd/Sd の path 着色のみ将来対応。

複数 phase 同時の場合、最初の ray spec の phase が大円弧着色に使われる（マーカーは全 phase 表示維持）。色付けする phase を変えるには map ray off してから別 phase で map ray <phase> する。

大規模ネットワークでの自動制御: - pair 数 > 80: depth-band ハイライト自動 OFF（マーカーは表示維持） - station 数 > 100: station ラベル自動 OFF - pair 数 > 80: turning/reflection の 深さラベル（“P 660 km” 等）自動 OFF（マーカー ◆ ◇ は表示）

明示制御の CLI:

map style                          # 現在値を表示
map style station-label on         # 観測点名を強制表示
map style station-label off        # 強制非表示
map style station-label auto       # 既定（100 station 以下のとき表示）
map style depth-label on|off|auto  # 深さ表示の同様制御（auto = pair 80 以下）
map style marker-label depth       # 既定。「P 660 km」「surface」表記
map style marker-label delta       # 「PP Δ=49°」表記（震央距離）
map style marker-label both        # 「PP 0 km / Δ=49°」（両方）

marker-label delta は PP/SS のような表面反射で特に有用（depth=0 で「surface」しか言わないところを震央距離で位置が読める）。

震源 ★ と観測点 ▽ の色

既定は研究室の慣習に合わせて 震源=赤 / 観測点=青 に統一（uniform モード）。後から色や配色モードを変えるオプションあり。

色を変えるだけなら *-color に色名を直接渡せる（自動で uniform モードに切替＋fill 設定）:

map style event-color purple        # 震源を紫の単色に
map style event-color '#ff7f0e'     # 16進指定
map style station-color teal        # 観測点をティールに

モードを明示的に変えたい時:

# 震源 ★
map style event-color uniform      # 既定。全震源を 1 色（event-fill）
map style event-color palette      # 震源ごとに palette 色（多イベント比較で便利）

# 観測点 ▽
map style station-color uniform    # 既定。全 station を 1 色（station-fill）
map style station-color palette    # 異なる station ごとに palette 色（≤10 station 向け）
map style station-color trace      # 初期実装の挙動。最初の trace 色を継承

uniform 時の fill 色を直接設定したい時（モードは触らない）:

map style event-fill '#d62728'     # 既定: Tableau 赤
map style station-fill '#1f77b4'   # 既定: Tableau 青

station の trace モードは「1 station = 1 trace」の単純ケースで trace 色との視覚リンクが欲しい時に使う。多数 station × 多成分のテレ地震解析では uniform、少数 station/震源の比較なら palette が読みやすい。

4.37. 経度ラップ（Plate Carrée）

地球は丸いので、Plate Carrée モードでパン/ズームを始めると画面外に出た領域が反対側から戻ってくる（spinning-globe feel）。例えば日本が画面左に消えていくと、アメリカが右から現れてくる。

  • 初期表示（自動フィット）ではラップ無効: 世界が一つだけ表示され、横長の canvas でも日本が二重に現れない。世界の輪郭枠（±180°）が見える
  • パン/ズーム開始でラップ有効: 世界を 360° ずつシフトしたコピーを画面に収まる範囲だけタイル描画。海岸線・震源・観測点・大円弧すべて同じ規則
  • AEQD には経度シームが無いため、ラップは常に適用されない
  • ダブルクリック / Esc で自動フィットへ戻る（ラップも自動的に解除）

4.38. 既知の制限

  • 大円弧は震源-観測点の単純な大円。波線（屈折/反射の幾何）は map ray / theory 併用で表示（§ 4.35）
  • AEQD は単一の中心点のみ（複数イベント時は自動 Plate Carrée fallback）
  • パン/ズームの undo は今のところなし（dblclick / Esc でリセット）

4.39. 仮想震源・観測点（波形ファイル不要）

波形ファイルなしで地図上に「仮想」の震源・観測点を配置して、theory + map ray を試せる。

map event add <lat> <lon> [name]   # 仮想震源追加（深さ既定 0）
map event depth <id> <km>          # 後で深さを設定
map event fm <id> <strike> <dip> <rake>  # 発震機構を付与 → ★を beachball に（off で解除）
map event list                     # real (波形ファイル由来) と virtual を区別して一覧
map event del <id>                 # ID（e1, e2, ...）で削除
map event clear                    # 仮想震源を全削除

map station add <lat> <lon> [name] # 仮想観測点追加
map station list / del / clear     # 同様

map place event                    # 次クリックで仮想震源を配置（Esc 中断）
map place station
map place off                      # モード解除

map pair real|virtual|all          # ペア構築モード切替（既定 all）
  • ID 体系: virtual は e1, e2, … / s1, s2, … の連番。削除後に再利用されないので、CLI 参照が session を通して安定する
  • 深さ未指定: 0 km（地表震源）扱いで theory が catalog を生成。後から map event depth e1 600 で変更可（自動で catalog clear）
  • 発震機構（map event fm）: 仮想イベントに strike/dip/rake（度）を付与すると ★ の代わりに下半球 beachball（GCMT レイヤと同じ描画）で表示。本震に公表メカニズム（例: SCARDEC / GCMT）を重ねる用途。map event fm <id> off で ★ に戻す。canvas / SVG 書出 / render.js すべて対応
  • クリック配置: AEQD / Plate Carrée の inverse で screen→geo 変換。geocentric → geodetic 変換も内部で実施
  • ペアリング:
    • all (既定): real↔︎real は sourceEntries 共有のペアのみ + virtual を含むペアは all-to-all
    • real: 波形ファイル由来のみ（virtual を完全除外）
    • virtual: virtual を含むペアのみ

theory <model> は波形ファイル由来のイベント + virtual イベント両方で catalog を構築する。波形ファイルがゼロ + 仮想 1 個でも動作する。

4.40. antipode と極通過大圏の取り扱い

震源と観測点が 正確に antipode（例: Tokyo (35, 140) ↔︎ (−35, −40)）の場合、大円弧は数学的に 無限通りある（任意の axis 周りに 180° 回転で antipode に到達）。Sympa!は deterministic に北極通過大圏弧を採用：

  • gcInterpolate / gcPointAt: sinDelta < 1e-6 で Rodrigues fallback に切替。axis = P₁ × NorthPole / |…| で北極通過の meridian-pair 大圏を選ぶ
  • AEQD project: cosC ≤ −1 + 1e-6 で antipode と判定し、disk 上端 (0, 180) を deterministic に返す
  • Plate Carrée splitAntimeridian: 隣接サンプルが |dLon| ≈ 180° かつ両側高緯度（>70°）の時、polyline を極点で切る（水平ワープ抑制）
  • drawMap Pass 1 の depth-band drawSeg にも同じ極通過判定

AEQD で antipode を表示すると disk 中心から真上に伸びる 1 本の vertical lineになる。Plate Carrée は 2 本の vertical line（震源側経度と antipode 側経度）に分離して描画。

4.41. SVG / PNG 出力

save <name>.svg [幅 高さ]      # 現在のビューをベクター SVG で保存
save <name>.png [幅 高さ]      # ラスター PNG（既定: 画面サイズ）
map save <name>.svg            # 互換エイリアス（`save` と同義）

論文・スライド向けの publication-quality 出力。現在の地図ビュー（投影・pan/zoom・レイヤ・タイント・プレート復元・断面図を含む）がそのまま SVG になる。

特徴: - SVG primitives（<path>/<polyline>/<polygon>/<text>/<line>/<rect>/<circle>/<ellipse>）のみ、<style> ブロックなし → Illustrator / Inkscape / Affinity Designer で個別要素を直接編集可 - 常に accurate-precision（画面を map precision coarse で軽くしていても、SVG は dense サンプル＋高精度 path） - 大規模データは描画解像度で間引き（波形・gather は画素ごと min/max エンベロープ）→ ファイル軽量 - antipode / 極通過大円・Plate Carrée の経度ラップ・AEQD 多イベント fallback も画面と同じ表現

Illustrator 互換（実機確認済み）: - 全 <text> に font-family="Helvetica" を直接指定する。指定なしだと Illustrator が ArialMT を代入し「一部の字体切り換え文字は…使用できません／白抜き文字として表示されます」警告（グリフが豆腐化）。カンマ区切り・CSS generic 名（system-ui 等）は不可 - clipPath 不使用（楕円/円フレームは塗りで表現）、paint-order 不使用（白ハローは「白文字＋色文字」の 2 枚重ね＝描画順で実現） - 色は #rrggbb（+ 必要なら fill-opacity）。hsl() / rgba() は Illustrator で無効化される - プレート名ラベルは SVG では英語（ASCII）（画面 canvas は日本語のまま）。Helvetica/Arial に CJK グリフが無いため。記号 Δ/°/−/· は Arial にあるのでそのまま、◆/◇ のみ ASCII 化

PNG: ヘッドレス（render.js）では @resvg/resvg-js を使用（npm i @resvg/resvg-js 済みなら自動、無ければ SVG にフォールバック）。

保存先: - cdlive 配下 → そのディレクトリに直接書き出す（showSaveFilePicker API） - snapshot mode（cd）→ ブラウザのダウンロード（~/Downloads） - ヘッドレス（render.js）→ 指定パスへ直接（~ 展開対応）

SVG → PDF: Illustrator / Inkscape で開いて PDF 保存（Sympa!は SVG のみ生成）。

4.42. 共有リンク / QRコード（ビューの再現）— share

share            # 現在のビューを再現する共有リンクを生成（URLをクリップボードにコピー＋表示）
share qr         # 同上＋QRコードをオーバーレイ表示（教室のスクリーンに出す用）

現在のビューを再現するコマンド列を URL のハッシュ（#s=…）に埋め込んだ共有リンクを作る。リンク（または share qr のQRコード）を配れば、受け手は同じ HTML を開くだけで同じビューを手元で再現できる。オフライン単一 HTML なので、リンク先の sympa-globe.html さえあれば追加インストール不要。授業提示の配布・追試・図の再現に使う。

仕組み: - リンクには、そのセッションで入力したコマンドのうち可搬なビュー操作だけ（map … / 投影 / レイヤ / 時間 / 比較 など）が入る。自前データの読み込み（r / read / load）・保存（save / write）・ディレクトリ操作（cd 等）・ヘルプは除外される。したがって、組み込みデータにもとづく提示は追加のデータ配布なしにどこでも再現でき、自分で読み込んだ観測データに依存するビューは再現対象外（そのデータは受け手の手元にないため）。 - 共有リンク（…/sympa-globe.html#s=…）で開くと、起動時にそのコマンド列を自動再生してビューを再現する。アドレスバーも share 実行時に共有 URL に更新されるので、ブックマーク／リロードでも同じビューに戻れる。 - QRコードは外部サービスを使わずオフラインでその場生成（誤り訂正レベル L、byte モード）。URL が長すぎる（概ね数百文字超）と QR 化できない場合があるが、その場合もリンク自体は使える。

⚠ 重要：マウス操作（ドラッグ回転・パン・ホイールズーム）は共有リンクに含まれない。共有リンクが再現するのは「入力したコマンド列」であり、マウスで地球を掴んで回す・つまんで動かす・ホイールで拡大縮小するといったマウス操作そのものは記録されない（受け手側では既定の向き・倍率のまま開かれる）。見ている地域・向きまで完全に一致させて共有したい場合は、その視点をコマンドで指定する（例: map rotate 138 36 ＝ 経度138°・緯度36°を正面に＝日本を中心にした向き）。map コマンドによる操作（投影・レイヤ・時間・比較など）は共有リンクに含まれる。

⚠ 重要：QR・他者への共有は「Web上のURLで開いているとき」だけ有効。共有リンクは、いま教材を開いている URL の末尾に #s=… を付けたものである。教材をローカルファイル（file://…/sympa-globe.html）で開いている場合、共有リンクも file://… になり、そのPC上でしか開けない（スマホでQRを読んでも、他の人にリンクを送っても「見つからない／読み込めない」となる。ファイルは各PCのローカルパスにあるため）。スマホでのQR読み取りや配布に使うには、教材をWebのURLで開くこと： - Sympa! Globe は GitHub Pages で公開済み（https://… の公開URLで開く。最も確実。誰でも・スマホでも開ける）。 - または同じフォルダで簡易サーバを立てる: python3 -m http.server 8000 を実行し、そのPCでは http://localhost:8000/sympa-globe.html、スマホからは同じWi-Fiで http://<そのPCのLAN IP>:8000/sympa-globe.html（localhost はそのPC自身専用なのでスマホからは不可）。 - share 実行時に file:// で開いていると、教材が上記の案内を自動で表示する。

使いどころ: 「このプレート復元のこの年代・この比較を見せたい」を1本のリンク/QRで共有 → 学生が各自の端末で同じ提示を再現。論文・資料で図の再現可能性を担保する導線としても使える。

4.43. プレート境界

map layer plate on            # プレート境界を表示
map layer plate off           # 非表示（既定）

データ源: Bird (2003) Geochemistry, Geophysics, Geosystems 4(3) “PB2002” モデル。fraxen/tectonicplates GitHub の GeoJSON を圧縮してインライン同梱（量子化 0.01° ≈ 1.1 km）。

色分け: - 🔴 赤系 (#cc3333): 沈み込み帯（subduction zone、PB2002 で Type=subduction タグの境界） - ⚪ 灰系 (#888888): その他（変換断層、中央海嶺、大陸衝突境界、大陸リフト、etc.）

PB2002 原典の 7 分類（CCB/CTF/CRB/OCB/OTF/OSR/SUB）は元 GeoJSON が subduction vs その他に集約済み。

位置関係: 海岸線（下）→ プレート境界（中）→ 大円弧（上）の z-order。大円弧が常に最上層に来るようになっている。

カスタマイズ: map style に plate-sub / plate-other キーは現状ない（次の小改善候補）。当面はソース既定色で運用。

SVG 出力: map save の出力にもそのまま反映される。

沈み込み帯の三角歯: subduction zone の境界線には自動的に三角歯（◀）が描画され、overriding plate 側を指す。配置間隔は 28 px（pixel space、zoom 非依存）。PB2002 の規約「subducting plate is on the right of segment direction」に従い、segment 進行方向の 左側に歯が向く。

4.44. プレート運動ベクトル

map layer motion on           # MORVEL56 速度ベクトルを表示
map layer motion off          # 非表示（既定）

各 PB2002 境界の中点に 1 本の矢印を描画。矢印は PlateA 相対 PlateB の速度方向を示し、長さは |v|（mm/yr）に比例。

研究グレード（DeMets/MORVEL/Bird の論文や教科書）の慣習に揃えた 単一矢印スタイルを採用。視覚的に「PlateB を fixed と見立てて PlateA がこう動く」と読み取る。境界タイプ（拡大/収束/変換）の判別は: 拡大は灰色境界線、収束は赤い境界線 + 三角歯、変換は灰色境界線で見分ける（矢印形状自体では区別しない）。

スケール: 0.18 px/(mm/yr)、min 4 px、max 38 px、stroke 2.4 px。Easter Island 等の超高速プレートが画面を埋めないようクランプあり。

将来の改善余地: plateMotionArrows() は境界 tangent (tanE / tanN) を返しているため、Caltech Tectonics の didactic スタイル（収束=単一、拡大/変換=両側）への切替もコード変更だけで実現可能。現在は研究論文標準に寄せた単一表示。

map style motion-model gsrm21      # NNR-GSRM v2.1 (Kreemer 2014、geodetic)
map style motion-model nuvel1a     # NNR-NUVEL-1A (DeMets 1994、classic geologic)
モデル 出典 カバー数 性格
NNR-MORVEL56 Argus, Gordon & DeMets 2011 G³ 52 PB2002 プレート 現行標準、地質学的（数 Myr 平均）
NNR-GSRM v2.1 Kreemer, Holt & Haines 2014 G³ 38 PB2002 プレート 測地学的（GPS 由来、~15 年平均）
NNR-NUVEL-1A DeMets et al. 1994 GRL 14 大プレートのみ 古典標準、磁気異常 3 Myr 平均

小規模モデル（NUVEL-1A は 14 プレートのみ）では PB2002 のマイクロプレート境界に矢印が出ない（モデルが知らないプレート）。これは仕様。

データ源詳細（MORVEL56）: Bird 2003 の 52 プレートコードに合わせてマッピング: - Bird AF（アフリカ統合）→ MORVEL nb（Nubia、代表値） - Bird SO（ソマリア）→ MORVEL sm - Bird YA（Yangtze）→ MORVEL yz - Bird NB（North Bismarck）→ MORVEL NB（case-sensitive）

制限: - Plate Carrée のみ で描画。AEQD では local (E, N) と pixel (x, y) のローカルフレームが一致しないため、現状未対応（将来 Jacobian で対応予定） - 矢印色は #2e5a88 ハードコード（m.style.motionColor で内部設定可、CLI 未公開） - 同符号速度ペアは 1 本（PB2002 では各境界が 1 つのfeatureなので自然に一意）

4.45. 古地理復元（プレート再構成）

GPlates 流の有限回転モデルで、任意の年代における大陸の位置を復元して海岸線として描画する。日本列島の成り立ちから超大陸ロディニアまで連続的に遡れる。

map plate-model Merdith2021         # モデル選択（6 種類）
map plate-age 200                   # 200 Ma に切替（茶色の線で古地理を描画）
map plate-age play 30               # 逆回し: 0 → max を 30 秒で再生（現在→過去）
map plate-age play 30 forward       # 順回し: max → 0 を 30 秒で再生（過去→現在）
map plate-age stop                  # 停止（Esc でも停止）
map plate-model off                 # モデル解除（現在の地図に戻る）

map proj ortho                      # 地球（globe）に切替 ← ロックはこの投影専用
map lock 南中国                     # 南中国を画面中央に固定して時代を追う
                                    #   （地球上の地塊クリックでも同じ。再クリック/⟲/map home で解除）

視点ロック（地塊追従、ortho 専用）: map proj ortho の地球で地塊をクリック（または map lock <地塊名>）すると、その地塊が常に画面中央に来るように固定される。map plate-age のタイムスライダーを動かすと、中心がその地塊に追従して移動するので、特定の地塊だけを見つめたまま大陸の離合集散を追える（例: 南中国がゴンドワナから分離して現在のアジアに集合するまで）。解除は同じ地塊の再クリック、右上の ⟲ ボタン、または map home。

古海洋プレート名ラベル（消失プレートの追跡）: トポロジー同梱モデル（Cao2024 / Matthews2016 / Merdith2021）でプレート復元中、消えた古太平洋プレートに名前ラベルが付く（クラ / イザナギ / ファラロン / フェニックス / アルク / パンサラッサ ＋現代の太平洋/ココス/ナスカ/フィリピン海/カリブ）。各プレートの復元ポリゴンの重心に置かれ、沈み込みで消滅すると自動で消える。教育的な見どころ: map proj ortho で日本付近を見ながら map plate-age を 100→50 Ma と動かすと、イザナギプレートが北西から日本下へ沈み込んで ~50 Ma に消える様子、続いて クラプレートが現れて消える様子が追える。map layer platelabels off で消灯。対話キャンバス・SVG 書き出し・render.js ヘッドレス出力すべてで表示される。

play のデフォルトは 逆回し（0 → max、現代の地図を過去に巻き戻す視点）。forward を付けると 順回し（max → 0、超大陸から現在の配置へ向かう視点）— 大陸衝突や離散の動きを「時系列で順に追う」ときに自然。

操作（マウス + CLI）:

  • 下端のスライダーを直接ドラッグすると age が即時更新（live preview）。
  • スライダー上のラベル: 左端「0 Ma」、右端「 Ma」、ハンドル中央に「<現在 Ma>（モデル名）」。

  • モデル選択時は 海岸線・プレート境界・地形・プレート運動の各レイヤを自動 OFF（present-day 地図と古地理が混在しないように）。再表示は map layer coastline on 等。

エンジン精度:

pyGPlates v1.0.0（公式 Python ライブラリ）との 数値レベル完全一致（6 モデル × 10 プレート × 11 年代 = 660/660 ケースで Δ < 0.01°）。最深の Cao2024 は深時代（1100–1800 Ma）でも全 71 実プレート × 6 年代 = 426/426 で Δ < 3.2×10⁻⁵°（機械精度）を確認（参照専用の plate 999＝実ジオメトリ非割当のアンカーは比較対象外）。実装上の要点:

  • Crossover-aware bracket: .rot ファイルの crossover entries（同年代で fixed plate が違う重複行）を、moving plate の履歴の中で最小出現回数の fixed plate を持つ chain で補間する規約に従う。Müller2019 と Merdith2021 で file-order の慣習が逆だが、この「最小出現回数ルール」は両者に通用する。
  • Plate-circuit interpolation: bracketing entries の fixed plate が異なる（かつ crossover entry が無い）場合は両端点を絶対参照系に変換してから SLERP。
  • Broken chain propagation: 親プレートが未定義／範囲外の場合は null を上方伝播し、描画時にその feature をスキップ（現代位置に幽霊大陸を描かない）。

フィーチャ妥当性:

GPML の fromAge / toAge を尊重。例: ハワイ諸島（fromAge ≈ 83 Ma）は 500 Ma では描画されない。-999 は「現在まで」のセンチネル。

Merdith2021 の深時代カバレッジ:

Merdith バンドルは 3 つの GPML を併合（ビルド時）— shapes_coastlines（現代の海岸線形状、fromAge ≤ 600 Ma）+ shapes_continents（大陸断片、fromAge ~3900 Ma まで）+ shapes_cratons（Archean 楯状地核、同様に超深時代まで）。併合しないと 600 Ma 以上でほぼオーストラリアしか表示されないので注意（shapes_coastlines 単独ではそうなる）。Sympa! ビルドはデフォルトで全 3 ファイル併合済（合計 4,056 features、~3.4 MB）。

600 Ma 付近で陸地が急増する理由:

Merdith 2021 の shapes_coastlines は 1,718 features が fromAge=600 Ma ちょうど にタグ付けされている（PALEOMAP / Müller の Phanerozoic 復元と同じ慣習）。これは意図的な設計で、「Phanerozoic（顕生代、≤ 541 Ma）以降は海洋底拡大の磁気異常から大陸位置が高精度に決まる → 現代海岸線形状を回転して使う」、「Neoproterozoic（≥ 600 Ma）は不確実性が大きい → 大陸 / クラトンの大まかな輪郭だけを示す」という二段構えになっているため。よって 601 Ma → 599 Ma で shapes_coastlines の 1,700+ features が一気に visibility に入り、見かけ上「陸地が急増」する。バグではなくデータセット側の表現（Merdith et al. 2021, ESR 214, §3.1 / Fig. 2 と本文）。物理的な陸地面積が突然増えるわけではない — 表示の粒度が切り替わる。

分類済みプレート境界:

Merdith2021（および将来 Matthews2016）には 時代対応のプレート境界トポロジーが同梱されている。map layer plate on で表示（モデル選択時は自動 ON）。

クラス 色 内容
sub 赤 + 三角歯 沈み込み帯（GPML SubductionZone）
mor 青 海嶺・大陸リフト（MidOceanRidge / ContinentalRift / ExtendedContinentalCrust）
trf 灰 トランスフォーム（Transform）
other 細灰 衝突帯など分類外

実装: pyGPlates で 5 Ma 刻みに resolve_topologies した結果をビルド時に JSON 化（201 スライス、~3 MB）。レンダラは指定年代に最近接のスライスに snap（5 Ma 内の補間はトポロジー的に正しくないので避けている）。歯の方向は pyGPlates の sub-segment winding に従う（overriding plate が線の左側）。

時代対応の速度ベクトル:

map layer motion on で各プレートの瞬間速度を矢印で描画。plateRotationVelocity が rotation engine の中心差分（dt=1 Ma）で計算 — 速度ベクトル用に別データは不要。モデル選択時に自動 ON。色・スケールは前述のプレート運動ベクトルと同じ（青、min 4 px / max 38 px、0.18 px/(mm/yr)）。map layer motion off で消せる。

閉じたプレート多角形:

map layer platepolys on で各プレートを薄塗り領域として描画（モデル選択時自動 ON）。色は plate ID の golden-angle HSL hash（同じプレートが時間を通して同じ色を保つ）、alpha=0.18 で coastlines と boundaries が見える透明度。Merdith は最大 46 polygons/slice、Matthews 同様。map layer platepolys off で消せる。

地塊ラベル（インデックス）:

map layer platelabels on で代表的な地塊・プレート名を地図上に表示（モデル選択時自動 ON）。色分け: 大陸地塊・クラトンは茶色、海洋プレートは黒。ラベルは各地塊の「現在位置の代表点」をそのプレートの有限回転で運んで配置するので、復元図の中で常に地塊の中心に乗る。重なりは大きい地塊優先で間引く。

ラベルの地塊セットはモデルごとに切り替わる。Phanerozoic 系モデル（Merdith2021 / Muller2019 / Matthews2016 / Scotese / Cao2024）は現代の大陸・プレート ID（北米・南米・ユーラシア…）を使う。一方 Li et al. 2013 RODINIA は前カンブリアのクラトンを独自 ID で持つため、専用のクラトン索引（ローレンシア・バルティカ・シベリア・アマゾニア・西アフリカ・コンゴ・カラハリ・北中国／南中国・タリム・インド・オーストラリア・東南極ほか計 16 地塊）を表示する。これがないと現代 ID が解決せず 530–1100 Ma でラベルが一切出ない（RODINIA だけの症状だった）。RODINIA を minAge（530 Ma）より新しい側で見たときは現代の地図を出すので、その範囲では現代大陸名に戻る。

Euler 極（stage poles）:

map layer eulerpoles on で各プレートの瞬間運動の Euler 極（rotation axis が地球表面と交わる点）を小白丸＋十字で描画（既定 OFF、特殊機能）。Euler 極まわりに小円軌道で動くので、極の位置を見るだけでプレートがどの方向へ動くかが幾何的に読める。複数プレートで同じ極になっていたら剛体的に一体運動している証拠（例: Merdith 200 Ma で SA/IND/AFR/AUS/ANT が同じ極＝Gondwana 結合の可視化）。

スライダー上の地質境界マーカー:

時代スライダーの下に主要な地質時代境界の刻みを表示（モデル範囲内のものだけ）: K-Pg (66 Ma) / P-Tr (252 Ma) / カンブリア (541 Ma) / クライオジェニアン (720 Ma) / トニアン (1000 Ma) / コロンビア (1600 Ma)。後 2 つは深い RODINIA・Cao2024 でのみ現れる。

アニメーションループ:

map plate-age play loop で繰り返し再生（端まで行ったら反対側へワープして続行）。play 60 forward loop のように秒数 + 方向 + loop を組み合わせ可。stop / Esc で停止。

SVG エクスポート:

map save で出力する SVG にも古地理レイヤすべてが含まれる: plate-recon 海岸線 / 分類済み境界（歯付き）/ 地塊・プレートラベル / 閉じた多角形 / 速度ベクトル。Illustrator で開いて編集可能。

制限:

  • アニメ再生は 0 → max / max → 0 の一方向スイープ（ループは未対応）。
  • Scotese / Li2013 は海岸線のみ（time-resolved topology を持たない、map layer plate 等は空表示）。Müller2019 / Matthews2016 / Merdith2021 / Cao2024 は海岸線復元＋境界トポロジー完全対応。

典型ワークフロー（論文用 1 枚、波形ファイル不要）:

map event add 38.1 142.9 "2011 Tohoku"   # 仮想震源（§ 4.39）
map station add 35.7 139.7 Tokyo          # 仮想観測点
theory PREM              # 走時カタログ構築
map ray P PcP S ScS      # 4 波線を表示
map style marker-label both
map proj aeqd
map save fig1_raypath.svg

4.46. 全球地質プロビンス（geological provinces）

全世界の地質区分（クラトン・楯状地・造山帯・島弧・付加体・海洋地殻 など）を塗り分ける教材レイヤ。Hasterok et al. (2022) の新しい全球地質プロビンス図（Fig.8）＝ 914 プロビンス。

map layer provinces on             # 全球地質プロビンスを表示（既定: 最終造山の年代で色分け）
map style prov-color age|type|crust   # 色分け: 年代 / 種別 / 大陸-海洋-遷移地殻
map style prov-alpha <0-1>         # 透過度（既定 0.6）
map style prov-recon off           # プレート復元中も現在位置に固定
map style prov-ocean on|off        # 海洋地殻プロビンスの海塗り（既定 off）
  • age（既定）＝ 最終造山の年代（Ma）で色分け ＝ Hasterok+2022 Fig.8「Last Orogeny」相当（新しい=淡紫/青 → 古い=褐色、0–3000 Ma カラーバー凡例つき）。クラトン核が暖色、縁辺の若い造山帯が寒色。年代値は Hasterok+2022 本文（§3.2.2・考察）記載の各造山系の代表年代（論文自身が「色は年代に粗く対応／長命な系を代表」と明記）。海洋地殻プロビンス（造山なし）の海塗りは既定オフ（map style prov-ocean on で淡青灰表示・凡例スウォッチも併出）。
  • プレート多角形の塗り（platepolys）と相互排他: provinces を on にすると、プレートモデル選択時に自動 ON される platepolys の半透明塗りは自動 OFF になる（両者が重なって見えづらいため）。プレート境界線（map layer plate）は残る。両方を敢えて重ねたいときだけ map layer platepolys on。
  • type＝ 15 種別（craton/shield/orogenic belt/volcanic arc/accretionary complex/oceanic crust …）＝ Fig.7 相当。crust＝ 大陸 / 海洋 / 遷移地殻。

プレート復元との連動: map plate-model <名>（Cao2024 等）+ map plate-age <Ma> でプレート復元中は、各プロビンスを所属プレート（重心で判定）で当時の古地理位置に剛体回転して表示する（超大陸 Pangea/Rodinia の集合・離散が、年代色のまま見える）。map style prov-recon off で復元せず現在位置に固定。

map plate-model Cao2024
map layer provinces on
map plate-age 320              # 320 Ma — プロビンスがパンゲア状の配置に集まる
  • プレート境界（map layer plate）・LIP・日本の地体構造/地質図と重ねて、全球テクトニクスの教材に。
  • ⚠ 配色は暫定（地質専門家の確認後に確定予定）。

出典: Hasterok, D., Halpin, J., Hand, M., Collins, A., Kreemer, C., Gard, M.G., & Glorie, S. (2022). New maps of global geological provinces and tectonic plates. Earth-Science Reviews, 231, 104069. doi:10.1016/j.earscirev.2022.104069（データ github.com/dhasterok/global_tectonics, GPL-3.0）。

6. トラブルシューティング・既知の制限

cd で「system folder block」と言われる

  • map tomo split / map split は閲覧向け（SVG 出力は単一タイントのみ・ブラウザ表示専用）
  • プレート復元アニメ再生は 0 → max / max → 0 の一方向スイープ（loop オプションあり、§ 4.45）
